Databricks는 데이터 엔지니어링, 데이터 웨어하우스, 머신러닝, 생성형 AI를 하나의 통합 플랫폼에서 처리할 수 있는 **레이크하우스(Lakehouse)** 아키텍처를 기반으로 합니다. 기존에는 데이터 레이크와 데이터 웨어하우스를 별도로 운영하며 데이터를 복사하고 동기화해야 했지만, Databricks는 이 두 가지를 하나의 스토리지 계층 위에 통합하여 운영 복잡성과 비용을 줄입니다.

## 플랫폼 아키텍처 개요

Databricks 플랫폼은 세 개의 계층으로 구성됩니다.

1. **스토리지 계층**: 클라우드 오브젝트 스토리지(S3, ADLS, GCS)에 Delta Lake 형식으로 데이터를 저장합니다.
2. **컴퓨트 계층**: Apache Spark 기반 클러스터, SQL Warehouse, 서버리스 컴퓨트가 쿼리와 작업을 실행합니다.
3. **거버넌스 계층**: Unity Catalog가 데이터 자산의 접근 제어, 계보(lineage), 감사(audit)를 중앙 관리합니다.

<Tabs>
  <Tab title="레이크하우스">
    ## 레이크하우스란

    레이크하우스는 데이터 레이크의 비용 효율적인 오브젝트 스토리지와 데이터 웨어하우스의 ACID 트랜잭션·스키마 적용 기능을 결합한 아키텍처입니다.

    **핵심 특징:**
    - 단일 데이터 사본으로 BI, ML, 실시간 분석을 모두 지원
    - Delta Lake 형식으로 ACID 트랜잭션 보장
    - 구조화·반구조화·비구조화 데이터 모두 처리
    - 오픈 포맷(Parquet, Delta)으로 벤더 종속성 최소화

    <Note>
      레이크하우스 아키텍처의 상세 내용은 [레이크하우스](/platform/lakehouse) 페이지에서 확인할 수 있습니다.
    </Note>
  </Tab>
  <Tab title="Delta Lake">
    ## Delta Lake

    Delta Lake는 Databricks가 개발한 오픈소스 스토리지 레이어로, 클라우드 오브젝트 스토리지 위에 신뢰성 있는 데이터 레이크를 구축합니다.

    **주요 기능:**
    - **ACID 트랜잭션**: 동시 읽기·쓰기 작업에서도 데이터 일관성 보장
    - **스키마 적용(Schema Enforcement)**: 잘못된 데이터가 테이블에 삽입되는 것을 방지
    - **스키마 진화(Schema Evolution)**: 기존 데이터를 재처리하지 않고 컬럼 추가·변경 가능
    - **타임 트래블(Time Travel)**: 과거 버전의 데이터에 쿼리 실행 가능
    - **DML 지원**: `UPDATE`, `DELETE`, `MERGE` 작업을 오브젝트 스토리지에서 직접 실행

    ```sql
    -- 타임 트래블로 과거 버전 조회
    SELECT * FROM my_table VERSION AS OF 10;

    -- 특정 시점의 데이터 조회
    SELECT * FROM my_table TIMESTAMP AS OF '2025-01-01 00:00:00';
    ```
  </Tab>
  <Tab title="Unity Catalog">
    ## Unity Catalog

    Unity Catalog는 Databricks의 통합 데이터 거버넌스 솔루션입니다. 메타스토어를 중앙화하여 여러 워크스페이스에 걸쳐 일관된 접근 제어와 데이터 계보를 제공합니다.

    **주요 기능:**
    - **3단계 네임스페이스**: `카탈로그.스키마.테이블` 형식으로 자산을 체계적으로 구성
    - **세분화된 권한 관리**: 테이블, 컬럼, 행 수준의 접근 제어
    - **데이터 계보 자동 추적**: 데이터가 어디서 왔고 어디로 흘러가는지 자동 기록
    - **데이터 검색**: 전사 데이터 자산을 검색하고 문서화

    <Note>
      Unity Catalog 설정 방법은 [Unity Catalog](/platform/unity-catalog) 페이지에서 상세히 다룹니다.
    </Note>
  </Tab>
  <Tab title="컴퓨트">
    ## 컴퓨트 유형

    Databricks는 워크로드에 따라 다양한 컴퓨트 옵션을 제공합니다.

    | 유형 | 용도 | 특징 |
    |---|---|---|
    | **All-Purpose 클러스터** | 대화형 분석, 노트북 | 지속 실행, 공유 가능 |
    | **Job 클러스터** | 자동화된 파이프라인 | 작업 시작·종료, 비용 최적화 |
    | **SQL Warehouse** | BI, SQL 분석 | 자동 스케일링, 서버리스 옵션 |
    | **서버리스 컴퓨트** | 빠른 시작, 관리 불필요 | 초 단위 빌링, 인프라 관리 불필요 |

    <Tip>
      개발·탐색 작업에는 All-Purpose 클러스터를, 프로덕션 파이프라인에는 Job 클러스터를 사용하면 비용을 크게 절감할 수 있습니다.
    </Tip>
  </Tab>
</Tabs>

## 클라우드 지원

Databricks는 세 개의 주요 클라우드 플랫폼에서 사용 가능하며, 각 클라우드의 네이티브 서비스와 긴밀하게 통합됩니다.

<AccordionGroup>
  <Accordion title="AWS (Amazon Web Services)">
    - 스토리지: Amazon S3
    - 컴퓨트: EC2 인스턴스 (GPU 포함)
    - 네트워킹: VPC, PrivateLink
    - ID 관리: IAM Role 기반 접근 제어
    - 마켓플레이스: AWS Marketplace를 통한 구독
  </Accordion>
  <Accordion title="Azure">
    - 스토리지: Azure Data Lake Storage Gen2
    - 컴퓨트: Azure VM (GPU 포함)
    - 네트워킹: VNet, Private Endpoint
    - ID 관리: Azure Active Directory (Entra ID)
    - 통합: Azure DevOps, Power BI, Synapse
  </Accordion>
  <Accordion title="Google Cloud Platform">
    - 스토리지: Google Cloud Storage
    - 컴퓨트: GCE 인스턴스 (GPU 포함)
    - 네트워킹: VPC, Private Service Connect
    - ID 관리: Google IAM, Workload Identity
    - 통합: BigQuery, Vertex AI, Looker
  </Accordion>
</AccordionGroup>

<Info>
  Databricks는 멀티클라우드 전략을 지원합니다. 동일한 Delta Lake 데이터와 Unity Catalog 거버넌스를 여러 클라우드에 걸쳐 사용할 수 있습니다.
</Info>
