Databricks는 데이터 플랫폼 위에서 생성형 AI(GenAI) 애플리케이션을 엔드-투-엔드로 구축할 수 있는 통합 환경을 제공합니다. 모델 선택부터 프롬프트 엔지니어링, 서빙, 모니터링까지 하나의 플랫폼에서 관리할 수 있어 데이터와 AI의 거버넌스를 일관되게 유지할 수 있습니다.

## GenAI 핵심 역량

<CardGroup cols={2}>
  <Card title="파운데이션 모델" icon="brain">
    Llama 3, Mistral, DBRX 등 오픈소스 파운데이션 모델을 Unity Catalog에서 관리하고 즉시 사용할 수 있습니다.
  </Card>
  <Card title="DBRX" icon="bolt">
    Databricks가 개발한 오픈소스 MoE(Mixture-of-Experts) 모델로, 동급 오픈소스 모델 대비 높은 추론 성능을 제공합니다.
  </Card>
  <Card title="Model Serving" icon="server">
    모델을 REST API 엔드포인트로 자동 배포하며, 오토스케일링과 A/B 테스트를 기본 지원합니다.
  </Card>
  <Card title="MLflow AI Gateway" icon="route">
    단일 API 인터페이스로 여러 LLM 공급자(OpenAI, Anthropic, Azure OpenAI 등)를 통합 관리합니다.
  </Card>
</CardGroup>

## Model Serving 엔드포인트 호출하기

Databricks Model Serving에 배포된 모델은 표준 REST API 또는 Python SDK로 호출할 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    ```python
    from databricks.sdk import WorkspaceClient
    from databricks.sdk.service.serving import ChatMessage, ChatMessageRole

    w = WorkspaceClient()

    response = w.serving_endpoints.query(
        name="databricks-dbrx-instruct",
        messages=[
            ChatMessage(
                role=ChatMessageRole.SYSTEM,
                content="당신은 Databricks 전문가입니다. 명확하고 간결하게 답변하세요."
            ),
            ChatMessage(
                role=ChatMessageRole.USER,
                content="Delta Lake와 Parquet의 차이점을 설명해주세요."
            ),
        ],
        max_tokens=512,
        temperature=0.1,
    )

    print(response.choices[0].message.content)
    ```
  </Tab>
  <Tab title="REST API (curl)">
    ```bash
    curl -X POST \
      "https://<workspace-url>/serving-endpoints/databricks-dbrx-instruct/invocations" \
      -H "Authorization: Bearer $DATABRICKS_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "messages": [
          {
            "role": "system",
            "content": "당신은 Databricks 전문가입니다."
          },
          {
            "role": "user",
            "content": "Delta Lake와 Parquet의 차이점을 설명해주세요."
          }
        ],
        "max_tokens": 512,
        "temperature": 0.1
      }'
    ```
  </Tab>
  <Tab title="OpenAI 호환 클라이언트">
    ```python
    from openai import OpenAI
    import os

    client = OpenAI(
        api_key=os.environ["DATABRICKS_TOKEN"],
        base_url=f"{os.environ['DATABRICKS_HOST']}/serving-endpoints"
    )

    response = client.chat.completions.create(
        model="databricks-dbrx-instruct",
        messages=[
            {"role": "system", "content": "당신은 Databricks 전문가입니다."},
            {"role": "user", "content": "Delta Lake와 Parquet의 차이점을 설명해주세요."}
        ],
        max_tokens=512,
        temperature=0.1,
    )

    print(response.choices[0].message.content)
    ```
  </Tab>
</Tabs>

<Note>
  `DATABRICKS_TOKEN` 및 `DATABRICKS_HOST` 환경 변수를 미리 설정해두면 코드에 자격 증명을 직접 입력하지 않아도 됩니다. Databricks Secrets를 사용하면 더욱 안전하게 관리할 수 있습니다.
</Note>

## GenAI 애플리케이션 구축 단계

<Steps>
  <Step title="모델 선택 및 평가">
    작업 유형(요약, Q&A, 코드 생성 등)과 응답 속도·비용 요구사항에 맞는 모델을 선택합니다. Databricks Playground에서 여러 모델을 비교 테스트해보세요.

    ```python
    # MLflow로 여러 모델 응답 품질 비교 평가
    import mlflow
    import pandas as pd

    eval_data = pd.DataFrame({
        "inputs": ["Delta Lake란 무엇인가요?", "Unity Catalog의 역할은?"],
        "ground_truth": ["Delta Lake는...", "Unity Catalog는..."]
    })

    with mlflow.start_run():
        results = mlflow.evaluate(
            model="endpoints:/databricks-dbrx-instruct",
            data=eval_data,
            targets="ground_truth",
            model_type="question-answering",
        )
    ```
  </Step>
  <Step title="프롬프트 엔지니어링">
    시스템 프롬프트와 사용자 프롬프트를 분리하여 역할, 응답 형식, 제약 조건을 명확히 정의합니다. 퓨샷(few-shot) 예시를 포함하면 출력 품질이 크게 향상됩니다.

    ```python
    SYSTEM_PROMPT = """당신은 Databricks 플랫폼 전문가입니다.
    다음 규칙을 따라 답변하세요:
    1. 기술적으로 정확한 정보만 제공합니다.
    2. 코드 예시는 Python 또는 SQL로 작성합니다.
    3. 모르는 내용은 솔직히 모른다고 답합니다.
    4. 답변은 한국어로 작성합니다."""

    def build_prompt(user_question: str, context: str = "") -> list:
        messages = [{"role": "system", "content": SYSTEM_PROMPT}]
        if context:
            messages.append({
                "role": "user",
                "content": f"참고 문서:\n{context}\n\n질문: {user_question}"
            })
        else:
            messages.append({"role": "user", "content": user_question})
        return messages
    ```
  </Step>
  <Step title="Model Serving 엔드포인트 배포">
    Unity Catalog에 등록된 모델 또는 외부 모델을 Model Serving 엔드포인트로 배포합니다. 프로비저닝 처리량(provisioned throughput)을 설정하면 지연 시간이 보장됩니다.

    ```python
    from databricks.sdk import WorkspaceClient
    from databricks.sdk.service.serving import (
        EndpointCoreConfigInput,
        ServedEntityInput,
    )

    w = WorkspaceClient()

    w.serving_endpoints.create(
        name="my-genai-endpoint",
        config=EndpointCoreConfigInput(
            served_entities=[
                ServedEntityInput(
                    entity_name="main.models.my_llm",
                    entity_version="3",
                    min_provisioned_throughput=0,
                    max_provisioned_throughput=100,
                    scale_to_zero_enabled=True,
                )
            ]
        ),
    )
    ```
  </Step>
  <Step title="모니터링 및 피드백 루프 구성">
    MLflow Tracing으로 LLM 호출을 추적하고, 응답 품질 지표를 대시보드에서 모니터링합니다. 사용자 피드백을 수집하여 프롬프트 또는 모델을 지속적으로 개선합니다.

    ```python
    import mlflow

    mlflow.set_experiment("/Shared/genai-monitoring")

    @mlflow.trace
    def generate_response(question: str) -> str:
        messages = build_prompt(question)
        response = w.serving_endpoints.query(
            name="my-genai-endpoint",
            messages=messages,
        )
        return response.choices[0].message.content
    ```
  </Step>
</Steps>

## 프롬프트 엔지니어링 핵심 기법

<AccordionGroup>
  <Accordion title="역할 부여 (Role Prompting)">
    시스템 메시지에 AI의 역할, 전문 분야, 응답 스타일을 명시합니다. 역할이 구체적일수록 일관된 품질의 응답을 얻을 수 있습니다.

    ```python
    system_message = """당신은 10년 경력의 데이터 엔지니어입니다.
    Apache Spark, Delta Lake, dbt에 정통하며
    대규모 데이터 파이프라인 설계 경험이 풍부합니다."""
    ```
  </Accordion>
  <Accordion title="퓨샷 학습 (Few-Shot Learning)">
    입력-출력 예시를 2~5개 제공하여 원하는 응답 형식과 수준을 모델에게 보여줍니다.

    ```python
    few_shot_examples = """
    예시 1:
    질문: Python에서 PySpark DataFrame을 만드는 방법은?
    답변: spark.createDataFrame(data, schema) 또는 spark.read.* 메서드를 사용합니다.

    예시 2:
    질문: Delta Lake에서 타임 트래블을 사용하는 방법은?
    답변: VERSION AS OF 또는 TIMESTAMP AS OF 구문을 SQL에서 사용하거나,
          .option("versionAsOf", 0)을 Python에서 사용합니다.
    """
    ```
  </Accordion>
  <Accordion title="체인 오브 소트 (Chain-of-Thought)">
    복잡한 추론이 필요한 질문에는 단계별로 생각하도록 유도합니다. "단계별로 설명해주세요" 또는 "Let's think step by step"을 프롬프트에 추가합니다.
  </Accordion>
  <Accordion title="출력 형식 제어">
    JSON, Markdown 표, 번호 목록 등 구조화된 출력이 필요한 경우, 정확한 형식을 프롬프트에 명시합니다. JSON 스키마를 직접 제공하면 더욱 안정적인 파싱이 가능합니다.

    ```python
    format_instruction = """
    반드시 다음 JSON 형식으로만 응답하세요:
    {
      "answer": "질문에 대한 답변",
      "confidence": "high|medium|low",
      "references": ["관련 문서 1", "관련 문서 2"]
    }
    """
    ```
  </Accordion>
</AccordionGroup>

<Tip>
  프로덕션 환경에서는 `temperature`를 0.0~0.2로 낮게 설정하여 일관성 있는 응답을 유도하세요. 창의적인 콘텐츠 생성이 목적이라면 0.7~1.0 범위를 사용합니다.
</Tip>

<Warning>
  모델 응답에 민감한 개인정보나 저작권 데이터가 포함되지 않도록 입출력 필터링 로직을 반드시 구현하세요. Databricks AI Security(DBAS) 가이드라인을 참고하여 책임 있는 AI 사용 정책을 수립하세요.
</Warning>
