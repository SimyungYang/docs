RAG(Retrieval-Augmented Generation)는 LLM의 응답에 기업 내부 문서나 최신 데이터를 실시간으로 주입하여 환각(hallucination)을 줄이고 답변 정확도를 높이는 아키텍처 패턴입니다. Databricks는 Vector Search, Model Serving, Unity Catalog를 통합하여 엔터프라이즈 수준의 RAG 시스템을 구축할 수 있는 완전한 환경을 제공합니다.

## RAG 아키텍처 개요

RAG 파이프라인은 크게 **오프라인 인덱싱**과 **온라인 추론**의 두 단계로 구성됩니다.

```
[오프라인 인덱싱]
원본 문서 → 청킹 → 임베딩 생성 → Vector Search 인덱스

[온라인 추론]
사용자 질문 → 질문 임베딩 → 유사 문서 검색 → LLM에 컨텍스트 주입 → 응답 생성
```

<Note>
  Databricks Vector Search는 Delta Lake 테이블과 직접 동기화되어, 원본 데이터가 업데이트될 때 인덱스도 자동으로 갱신됩니다. 별도의 ETL 파이프라인 없이 항상 최신 상태를 유지할 수 있습니다.
</Note>

## 단계별 RAG 파이프라인 구축

<Steps>
  <Step title="문서 로드 및 청킹">
    PDF, HTML, Markdown 등 다양한 형식의 문서를 로드하고, 검색과 컨텍스트 활용에 적합한 크기로 분할합니다. 청크 크기는 일반적으로 512~1024 토큰이 적합하며, 문단이나 섹션 경계를 기준으로 분할하면 의미 단위가 보존됩니다.

    ```python
    from langchain.document_loaders import PyPDFLoader, DirectoryLoader
    from langchain.text_splitter import RecursiveCharacterTextSplitter
    import pandas as pd

    # 문서 로드
    loader = DirectoryLoader(
        "/Volumes/main/docs/raw_documents/",
        glob="**/*.pdf",
        loader_cls=PyPDFLoader
    )
    documents = loader.load()

    # 청킹
    text_splitter = RecursiveCharacterTextSplitter(
        chunk_size=1000,
        chunk_overlap=200,
        separators=["\n\n", "\n", "。", ".", " ", ""],
        length_function=len,
    )
    chunks = text_splitter.split_documents(documents)

    # Delta Lake 테이블로 저장
    chunks_df = pd.DataFrame([
        {
            "chunk_id": i,
            "source": doc.metadata.get("source", ""),
            "page": doc.metadata.get("page", 0),
            "content": doc.page_content,
        }
        for i, doc in enumerate(chunks)
    ])

    spark_df = spark.createDataFrame(chunks_df)
    spark_df.write.format("delta").mode("overwrite").saveAsTable("main.rag.document_chunks")
    ```
  </Step>
  <Step title="임베딩 생성">
    각 청크에 대한 벡터 임베딩을 생성하여 Delta Lake 테이블에 저장합니다. Databricks에서 제공하는 임베딩 모델 엔드포인트를 사용하면 별도 인프라 없이 임베딩을 생성할 수 있습니다.

    ```python
    from databricks.sdk import WorkspaceClient
    import numpy as np

    w = WorkspaceClient()

    def get_embeddings(texts: list[str], batch_size: int = 100) -> list[list[float]]:
        """Databricks 임베딩 엔드포인트 호출"""
        all_embeddings = []
        for i in range(0, len(texts), batch_size):
            batch = texts[i:i + batch_size]
            response = w.serving_endpoints.query(
                name="databricks-gte-large-en",
                input=batch,
            )
            embeddings = [item.embedding for item in response.data]
            all_embeddings.extend(embeddings)
        return all_embeddings

    # Delta 테이블에서 청크 로드
    chunks_df = spark.table("main.rag.document_chunks").toPandas()

    # 임베딩 생성
    embeddings = get_embeddings(chunks_df["content"].tolist())
    chunks_df["embedding"] = embeddings

    # 임베딩 포함 테이블 저장
    spark.createDataFrame(chunks_df).write \
        .format("delta") \
        .mode("overwrite") \
        .saveAsTable("main.rag.document_chunks_with_embeddings")
    ```

    <Tip>
      한국어 문서에는 `multilingual-e5-large` 또는 `paraphrase-multilingual-mpnet-base-v2` 임베딩 모델을 사용하면 더 나은 검색 품질을 얻을 수 있습니다.
    </Tip>
  </Step>
  <Step title="Vector Search 인덱스 생성">
    임베딩이 포함된 Delta Lake 테이블을 기반으로 Vector Search 인덱스를 생성합니다. Delta Sync 방식을 사용하면 원본 테이블 변경사항이 자동으로 인덱스에 반영됩니다.

    ```python
    from databricks.sdk import WorkspaceClient
    from databricks.sdk.service.vectorsearch import (
        VectorIndexType,
        DeltaSyncVectorIndexSpecRequest,
        EmbeddingSourceColumn,
        PipelineType,
    )

    w = WorkspaceClient()

    # Vector Search 엔드포인트 생성 (최초 1회)
    w.vector_search_endpoints.create_endpoint_and_wait(
        name="rag-vector-search-endpoint",
        endpoint_type="STANDARD",
    )

    # Delta Sync 인덱스 생성
    index = w.vector_search_indexes.create_index(
        name="main.rag.document_index",
        endpoint_name="rag-vector-search-endpoint",
        primary_key="chunk_id",
        index_type=VectorIndexType.DELTA_SYNC,
        delta_sync_index_spec=DeltaSyncVectorIndexSpecRequest(
            source_table="main.rag.document_chunks_with_embeddings",
            pipeline_type=PipelineType.TRIGGERED,
            embedding_vector_columns=[
                EmbeddingSourceColumn(
                    name="embedding",
                    embedding_dimension=1024,
                )
            ],
        ),
    )

    print(f"인덱스 생성 완료: {index.name}")
    ```
  </Step>
  <Step title="유사 문서 검색">
    사용자 질문을 동일한 임베딩 모델로 변환한 뒤 Vector Search 인덱스에서 가장 유사한 청크를 검색합니다.

    ```python
    from databricks.sdk import WorkspaceClient

    w = WorkspaceClient()

    def retrieve_relevant_chunks(
        query: str,
        num_results: int = 5,
        score_threshold: float = 0.7,
    ) -> list[dict]:
        """사용자 질문과 관련된 문서 청크를 검색합니다."""

        # 질문 임베딩 생성
        query_embedding = get_embeddings([query])[0]

        # Vector Search에서 유사 문서 검색
        results = w.vector_search_indexes.query_index(
            index_name="main.rag.document_index",
            columns=["chunk_id", "source", "page", "content"],
            query_vector=query_embedding,
            num_results=num_results,
            filters={"score": {"$gte": score_threshold}},
        )

        chunks = []
        for result in results.result.data_array:
            chunk_id, source, page, content, score = result
            chunks.append({
                "content": content,
                "source": source,
                "page": page,
                "relevance_score": score,
            })

        return chunks
    ```
  </Step>
  <Step title="LLM으로 응답 생성">
    검색된 문서 청크를 컨텍스트로 구성하여 LLM에 전달하고 최종 응답을 생성합니다.

    ```python
    from databricks.sdk import WorkspaceClient
    from databricks.sdk.service.serving import ChatMessage, ChatMessageRole

    w = WorkspaceClient()

    def generate_rag_response(user_question: str) -> dict:
        """RAG 파이프라인으로 질문에 답변합니다."""

        # 1. 관련 문서 검색
        relevant_chunks = retrieve_relevant_chunks(user_question, num_results=5)

        if not relevant_chunks:
            return {
                "answer": "관련 문서를 찾을 수 없습니다. 질문을 다시 구체적으로 입력해주세요.",
                "sources": [],
            }

        # 2. 컨텍스트 구성
        context = "\n\n---\n\n".join([
            f"[출처: {chunk['source']}, 페이지: {chunk['page']}]\n{chunk['content']}"
            for chunk in relevant_chunks
        ])

        # 3. LLM 호출
        system_prompt = """당신은 기업 내부 문서를 기반으로 질문에 답변하는 AI 어시스턴트입니다.
        반드시 제공된 참고 문서의 내용만을 바탕으로 답변하세요.
        문서에 없는 내용은 "제공된 문서에서 해당 정보를 찾을 수 없습니다."라고 답하세요."""

        user_message = f"""참고 문서:
{context}

질문: {user_question}"""

        response = w.serving_endpoints.query(
            name="databricks-dbrx-instruct",
            messages=[
                ChatMessage(role=ChatMessageRole.SYSTEM, content=system_prompt),
                ChatMessage(role=ChatMessageRole.USER, content=user_message),
            ],
            max_tokens=1024,
            temperature=0.1,
        )

        return {
            "answer": response.choices[0].message.content,
            "sources": [
                {"source": c["source"], "page": c["page"], "score": c["relevance_score"]}
                for c in relevant_chunks
            ],
        }

    # 실행 예시
    result = generate_rag_response("Databricks Unity Catalog에서 데이터 접근 권한을 설정하는 방법은?")
    print(result["answer"])
    print("\n참고 출처:", result["sources"])
    ```
  </Step>
</Steps>

## LangChain으로 통합 구현하기

LangChain을 사용하면 위의 단계를 간결한 체인으로 구성할 수 있습니다.

```python
from langchain_community.vectorstores import DatabricksVectorSearch
from langchain_community.embeddings import DatabricksEmbeddings
from langchain_community.chat_models import ChatDatabricks
from langchain.chains import RetrievalQA
from langchain.prompts import PromptTemplate

# 임베딩 모델 설정
embeddings = DatabricksEmbeddings(endpoint="databricks-gte-large-en")

# Vector Search 연결
vector_store = DatabricksVectorSearch(
    endpoint="rag-vector-search-endpoint",
    index_name="main.rag.document_index",
    embedding=embeddings,
    text_column="content",
    columns=["source", "page"],
)

# LLM 설정
llm = ChatDatabricks(
    endpoint="databricks-dbrx-instruct",
    max_tokens=1024,
    temperature=0.1,
)

# 프롬프트 템플릿
prompt_template = """다음 참고 문서를 바탕으로 질문에 답변하세요.
문서에 없는 내용은 답변하지 마세요.

참고 문서:
{context}

질문: {question}

답변:"""

PROMPT = PromptTemplate(
    template=prompt_template,
    input_variables=["context", "question"]
)

# RAG 체인 구성
rag_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=vector_store.as_retriever(search_kwargs={"k": 5}),
    chain_type_kwargs={"prompt": PROMPT},
    return_source_documents=True,
)

# 질의
result = rag_chain.invoke({"query": "Delta Live Tables란 무엇인가요?"})
print(result["result"])
```

## 검색 품질 최적화

<AccordionGroup>
  <Accordion title="하이브리드 검색 (BM25 + 벡터 검색)">
    키워드 기반 BM25 검색과 의미 기반 벡터 검색을 결합하면 검색 재현율을 높일 수 있습니다. 특히 제품명, 코드, 고유명사가 포함된 쿼리에서 효과적입니다.

    ```python
    from databricks.sdk.service.vectorsearch import QueryType

    # 하이브리드 검색 (벡터 + 키워드)
    results = w.vector_search_indexes.query_index(
        index_name="main.rag.document_index",
        columns=["chunk_id", "content", "source"],
        query_vector=query_embedding,
        query_text=user_question,  # 키워드 검색 병행
        num_results=10,
        query_type=QueryType.HYBRID,
    )
    ```
  </Accordion>
  <Accordion title="메타데이터 필터링">
    문서 유형, 날짜, 부서 등 메타데이터로 검색 범위를 좁혀 정확도를 높입니다.

    ```python
    # 특정 부서 문서만 검색
    results = w.vector_search_indexes.query_index(
        index_name="main.rag.document_index",
        columns=["chunk_id", "content", "source", "department"],
        query_vector=query_embedding,
        num_results=5,
        filters={"department": {"$eq": "engineering"}},
    )
    ```
  </Accordion>
  <Accordion title="재순위화 (Reranking)">
    초기 검색 결과에 크로스-인코더 모델을 적용하여 관련성이 가장 높은 문서를 상위에 배치합니다. 검색 품질은 높아지지만 응답 지연이 증가할 수 있습니다.

    ```python
    from sentence_transformers import CrossEncoder

    reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")

    def rerank_results(query: str, chunks: list[dict], top_k: int = 3) -> list[dict]:
        pairs = [(query, chunk["content"]) for chunk in chunks]
        scores = reranker.predict(pairs)
        ranked = sorted(
            zip(chunks, scores),
            key=lambda x: x[1],
            reverse=True
        )
        return [chunk for chunk, _ in ranked[:top_k]]
    ```
  </Accordion>
  <Accordion title="청킹 전략 최적화">
    고정 크기 청킹 외에 문서 구조를 활용한 계층적 청킹을 사용하면 검색 품질이 향상됩니다. 제목-내용 쌍을 함께 저장하면 컨텍스트 손실을 줄일 수 있습니다.

    ```python
    # 계층적 청킹: 섹션 제목을 각 청크에 포함
    def hierarchical_chunk(document: str, section_title: str) -> list[str]:
        splitter = RecursiveCharacterTextSplitter(chunk_size=800, chunk_overlap=100)
        raw_chunks = splitter.split_text(document)
        # 각 청크에 섹션 제목 접두어 추가
        return [f"섹션: {section_title}\n\n{chunk}" for chunk in raw_chunks]
    ```
  </Accordion>
</AccordionGroup>

## RAG 평가 지표

<CardGroup cols={3}>
  <Card title="Faithfulness" icon="check-circle">
    LLM 응답이 제공된 컨텍스트에 근거하는 비율. RAGAS 라이브러리로 자동 측정 가능합니다.
  </Card>
  <Card title="Answer Relevancy" icon="target">
    생성된 답변이 질문과 얼마나 관련 있는지를 측정합니다. 주제를 벗어난 응답을 탐지합니다.
  </Card>
  <Card title="Context Recall" icon="database">
    정답 생성에 필요한 정보가 검색된 컨텍스트에 포함된 비율로, 검색 품질을 측정합니다.
  </Card>
</CardGroup>

```python
# RAGAS로 RAG 파이프라인 평가
from ragas import evaluate
from ragas.metrics import faithfulness, answer_relevancy, context_recall
from datasets import Dataset

eval_dataset = Dataset.from_dict({
    "question": ["Delta Lake란?", "Unity Catalog 역할은?"],
    "answer": [rag_chain.invoke(q)["result"] for q in questions],
    "contexts": [[c["content"] for c in retrieve_relevant_chunks(q)] for q in questions],
    "ground_truth": ["Delta Lake는 오픈소스 스토리지 레이어...", "Unity Catalog는 통합 거버넌스..."],
})

result = evaluate(eval_dataset, metrics=[faithfulness, answer_relevancy, context_recall])
print(result)
```

<Warning>
  프로덕션 RAG 시스템에는 반드시 입력 검증 및 PII(개인 식별 정보) 필터링 로직을 추가하세요. 사용자 질문에 포함된 민감한 정보가 LLM API 공급자로 전송될 수 있습니다.
</Warning>
