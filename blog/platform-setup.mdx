Databricks 워크스페이스를 처음 생성하면 기본 설정으로도 작동하지만, 프로덕션 환경에서는 보안, 거버넌스, 비용 관리를 위한 추가 설정이 필수입니다. 이 가이드는 엔터프라이즈 수준의 워크스페이스를 구성하기 위한 단계별 체크리스트를 제공합니다.

<Note>
  이 가이드는 AWS, Azure, GCP 모두에 적용되는 플랫폼 공통 설정을 다룹니다. 클라우드별 세부 설정(VPC, VNet, 서비스 계정 등)은 각 클라우드 공급자의 Databricks 설명서를 함께 참고하세요.
</Note>

## 프로덕션 워크스페이스 구성 단계

<Steps>
  <Step title="Unity Catalog 설정">
    Unity Catalog는 Databricks의 통합 데이터 거버넌스 레이어입니다. 계정 수준에서 Metastore를 먼저 생성하고, 워크스페이스에 연결해야 합니다.

    **Metastore 생성 (Databricks 계정 관리자 권한 필요):**

    ```bash
    # Databricks CLI로 Metastore 생성
    databricks metastores create \
      --name "production-metastore" \
      --region "ap-northeast-2" \
      --storage-root "s3://my-company-unity-catalog/metastore"
    ```

    **워크스페이스에 Metastore 연결:**

    ```bash
    databricks metastores assign \
      --workspace-id 1234567890 \
      --metastore-id <metastore-id> \
      --default-catalog-name "main"
    ```

    **카탈로그 계층 구조 설계 예시:**

    ```
    Unity Catalog (Metastore)
    ├── main (카탈로그 — 프로덕션 데이터)
    │   ├── raw        (원시 데이터 스키마)
    │   ├── silver     (정제 데이터 스키마)
    │   ├── gold       (집계 데이터 스키마)
    │   └── models     (ML 모델 스키마)
    ├── dev (카탈로그 — 개발/테스트 데이터)
    │   └── sandbox
    └── shared (카탈로그 — 팀 간 공유 데이터)
        └── reference
    ```

    <Tip>
      카탈로그는 환경(prod/dev/stg)별로 분리하고, 스키마는 도메인 또는 팀별로 분리하는 것이 권장됩니다. 이렇게 하면 환경 간 데이터 격리와 팀별 접근 제어가 용이합니다.
    </Tip>
  </Step>
  <Step title="네트워크 보안 설정">
    프로덕션 환경에서는 퍼블릭 인터넷 노출을 최소화하고, 기업 네트워크와의 안전한 연결을 구성해야 합니다.

    **Private Link / Private Endpoint 활성화:**

    ```bash
    # Azure의 경우 Private Endpoint 설정 (Terraform 예시)
    resource "azurerm_private_endpoint" "databricks_ui" {
      name                = "databricks-ui-private-endpoint"
      location            = var.location
      resource_group_name = var.resource_group_name
      subnet_id           = azurerm_subnet.private_endpoint_subnet.id

      private_service_connection {
        name                           = "databricks-ui-connection"
        private_connection_resource_id = azurerm_databricks_workspace.main.id
        subresource_names              = ["databricks_ui_api"]
        is_manual_connection           = false
      }
    }
    ```

    **IP 접근 제어 목록(IP ACL) 설정:**

    ```python
    # Databricks SDK로 IP ACL 설정
    from databricks.sdk import AccountClient

    a = AccountClient()

    a.network_connectivity.create(
        network_connectivity_config_name="prod-ip-acl",
        region="ap-northeast-2",
    )
    ```

    **권장 네트워크 아키텍처:**

    | 컴포넌트 | 권장 설정 |
    |---------|---------|
    | Control Plane 접근 | Private Link 사용 |
    | Data Plane 서브넷 | 전용 VPC/VNet의 프라이빗 서브넷 |
    | NAT Gateway | 아웃바운드 인터넷 트래픽 통제 |
    | 방화벽 | 허용된 도메인만 아웃바운드 허용 |
  </Step>
  <Step title="ID 및 접근 관리 (IAM) 설정">
    사용자와 서비스 주체(Service Principal)에 대한 최소 권한 원칙 기반 접근 제어를 설정합니다.

    **SCIM을 통한 사용자 프로비저닝 (Entra ID/Okta 연동):**

    ```bash
    # Entra ID SCIM 연동 설정 (계정 관리자 콘솔에서 수행)
    # Settings → User provisioning → SCIM token 생성
    databricks accounts service-principals create \
      --display-name "scim-provisioner" \
      --entitlements allow-cluster-create
    ```

    **서비스 주체(Service Principal) 생성 및 권한 부여:**

    ```python
    from databricks.sdk import AccountClient
    from databricks.sdk.service.iam import ComplexValue, RoleName

    a = AccountClient()

    # 서비스 주체 생성
    sp = a.service_principals.create(display_name="prod-pipeline-sp")

    # 워크스페이스에 권한 부여
    a.workspace_assignment.update(
        workspace_id=1234567890,
        principal_id=sp.id,
        permissions=[RoleName.USER],
    )
    ```

    **Unity Catalog 권한 계층 설정 (SQL):**

    ```sql
    -- 카탈로그 수준 권한
    GRANT USE CATALOG ON CATALOG main TO `data-engineers`;
    GRANT CREATE CATALOG ON METASTORE TO `catalog-admins`;

    -- 스키마 수준 권한
    GRANT USE SCHEMA, CREATE TABLE ON SCHEMA main.silver TO `data-engineers`;
    GRANT SELECT ON SCHEMA main.gold TO `data-analysts`;

    -- 테이블 수준 세분화 권한
    GRANT SELECT ON TABLE main.gold.sales_summary TO `sales-team`;

    -- 행 수준 보안 (Row-Level Security)
    CREATE ROW FILTER policy_region_filter ON main.gold.sales_detail
      USING (region = current_user_region());

    ALTER TABLE main.gold.sales_detail
      SET ROW FILTER policy_region_filter ON ();
    ```
  </Step>
  <Step title="컴퓨트 정책 및 클러스터 관리">
    클러스터 유형, 크기, 자동 종료 등을 정책으로 강제하여 비용 낭비와 보안 위험을 방지합니다.

    **클러스터 정책 생성 (JSON):**

    ```json
    {
      "spark_version": {
        "type": "allowlist",
        "values": ["14.3.x-scala2.12", "15.4.x-scala2.12"],
        "defaultValue": "15.4.x-scala2.12"
      },
      "autotermination_minutes": {
        "type": "range",
        "minValue": 10,
        "maxValue": 120,
        "defaultValue": 30
      },
      "node_type_id": {
        "type": "allowlist",
        "values": ["m5.xlarge", "m5.2xlarge", "m5.4xlarge"]
      },
      "num_workers": {
        "type": "range",
        "minValue": 1,
        "maxValue": 10
      },
      "data_security_mode": {
        "type": "fixed",
        "value": "SINGLE_USER"
      }
    }
    ```

    ```python
    from databricks.sdk import WorkspaceClient
    import json

    w = WorkspaceClient()

    policy = w.cluster_policies.create(
        name="production-standard-policy",
        definition=json.dumps(cluster_policy_config),
        description="프로덕션 환경의 표준 클러스터 정책",
    )
    ```

    <Warning>
      `data_security_mode`를 `SINGLE_USER`로 고정하면 Unity Catalog의 행/열 수준 보안이 정상적으로 적용됩니다. `LEGACY_PASSTHROUGH` 또는 `NONE` 모드에서는 일부 보안 기능이 작동하지 않을 수 있습니다.
    </Warning>
  </Step>
  <Step title="비밀(Secrets) 관리 설정">
    데이터베이스 연결 정보, API 키, 서비스 계정 자격 증명 등을 Databricks Secrets에 안전하게 저장합니다.

    ```bash
    # Secret Scope 생성 (Databricks CLI)
    databricks secrets create-scope prod-secrets

    # Azure Key Vault 연동 (AKV 기반 Scope)
    databricks secrets create-scope \
      --scope prod-kv-secrets \
      --scope-backend-type AZURE_KEYVAULT \
      --resource-id /subscriptions/.../resourceGroups/.../providers/Microsoft.KeyVault/vaults/prod-kv \
      --dns-name https://prod-kv.vault.azure.net/

    # Secret 추가
    databricks secrets put-secret prod-secrets db-password \
      --string-value "your-database-password"
    ```

    **노트북에서 Secret 사용:**

    ```python
    # Secret 조회 (값은 노출되지 않음)
    db_password = dbutils.secrets.get(scope="prod-secrets", key="db-password")

    # JDBC 연결에 활용
    jdbc_url = f"jdbc:postgresql://prod-db.example.com:5432/mydb"
    df = spark.read.format("jdbc") \
        .option("url", jdbc_url) \
        .option("dbtable", "public.sales") \
        .option("user", "readonly_user") \
        .option("password", db_password) \
        .load()
    ```
  </Step>
  <Step title="모니터링 및 감사 설정">
    워크스페이스의 모든 활동을 로깅하고, 이상 징후를 감지할 수 있는 모니터링 체계를 구축합니다.

    ```python
    # Audit Log를 Delta Lake로 수집하는 파이프라인
    # (Databricks 계정 설정에서 Audit Log 전송 대상 설정 후)

    from pyspark.sql import functions as F

    audit_logs = spark.readStream \
        .format("cloudFiles") \
        .option("cloudFiles.format", "json") \
        .load("s3://my-audit-logs/databricks/")

    # 중요 이벤트 필터링 및 저장
    critical_events = audit_logs.filter(
        F.col("actionName").isin([
            "create", "delete", "changePermissions",
            "login", "logout", "tokenCreated"
        ])
    )

    critical_events.writeStream \
        .format("delta") \
        .option("checkpointLocation", "/mnt/checkpoints/audit-logs") \
        .outputMode("append") \
        .toTable("main.security.audit_logs")
    ```

    ```sql
    -- 비정상 접근 패턴 탐지 쿼리
    SELECT
        userIdentity.email,
        COUNT(*) AS failed_logins,
        MIN(timestamp) AS first_attempt,
        MAX(timestamp) AS last_attempt
    FROM main.security.audit_logs
    WHERE
        actionName = 'login'
        AND response.statusCode != 200
        AND timestamp > current_timestamp() - INTERVAL 1 HOUR
    GROUP BY userIdentity.email
    HAVING COUNT(*) > 5
    ORDER BY failed_logins DESC;
    ```
  </Step>
</Steps>

## 프로덕션 준비 체크리스트

<AccordionGroup>
  <Accordion title="보안 체크리스트">
    다음 항목을 확인하여 보안 설정을 완료하세요.

    - [ ] Unity Catalog Metastore가 설정되고 워크스페이스에 연결됨
    - [ ] Private Link / Private Endpoint가 활성화됨
    - [ ] IP 접근 제어 목록(IP ACL)이 설정됨
    - [ ] 모든 클러스터에 `SINGLE_USER` 또는 `USER_ISOLATION` 보안 모드 적용
    - [ ] 서비스 주체(Service Principal)는 최소 권한 원칙으로 설정됨
    - [ ] 데이터베이스 자격 증명은 Secrets에만 저장하고 코드에 하드코딩 금지
    - [ ] Audit Log 수집이 활성화되고 Delta Lake로 전송됨
    - [ ] 관리자 계정에 MFA(다단계 인증)가 활성화됨
  </Accordion>
  <Accordion title="거버넌스 체크리스트">
    - [ ] 카탈로그-스키마-테이블 계층 구조가 팀/도메인/환경에 맞게 설계됨
    - [ ] 민감 데이터 테이블에 열 마스킹(Column Masking) 정책이 적용됨
    - [ ] 필요한 경우 행 수준 보안(Row-Level Security)이 구성됨
    - [ ] 데이터 분류 태그(PII, Confidential 등)가 테이블에 부착됨
    - [ ] 데이터 계보(Lineage) 추적이 활성화됨
    - [ ] 그룹 기반 접근 제어로 개인 단위 권한 부여 최소화
  </Accordion>
  <Accordion title="운영 안정성 체크리스트">
    - [ ] 모든 인터랙티브 클러스터에 자동 종료(Auto-termination) 설정됨 (최대 120분)
    - [ ] 클러스터 정책으로 허용 인스턴스 유형 및 크기 제한됨
    - [ ] Delta Live Tables 또는 Databricks Workflows로 파이프라인이 자동화됨
    - [ ] 중요 파이프라인 실패 시 이메일/Slack 알림이 설정됨
    - [ ] 워크스페이스 태그(Team, CostCenter, Environment)가 일관되게 적용됨
    - [ ] 주요 Delta 테이블에 `OPTIMIZE` 및 `VACUUM` 작업이 스케줄됨
  </Accordion>
</AccordionGroup>

<Info>
  Databricks는 공식 보안 가이드 "Databricks Security Best Practices"를 제공합니다. 금융, 헬스케어, 공공 분야는 해당 분야의 컴플라이언스 가이드(HIPAA, FedRAMP 등)도 함께 확인하세요.
</Info>
