Databricks를 처음 시작한다면 이 페이지에서 안내하는 순서대로 따라오세요. 계정 생성부터 첫 번째 PySpark 코드 실행까지 약 30분이면 완료할 수 있습니다. 설치가 필요한 소프트웨어는 없으며, 웹 브라우저만 있으면 됩니다.

<Info>
  이 가이드는 Databricks Community Edition 또는 유료 계정 모두에서 사용할 수 있습니다. Community Edition은 무료이며 학습 목적으로 충분합니다. 단, 서버리스 컴퓨트와 Unity Catalog 일부 기능은 유료 계정에서만 사용 가능합니다.
</Info>

## 사전 요건

시작하기 전에 아래 항목을 확인하세요.

- 유효한 이메일 주소 (기업 이메일 권장)
- 최신 버전의 Chrome, Firefox, Edge, 또는 Safari
- 클라우드 계정 (유료 플랜의 경우): AWS, Azure, 또는 GCP 계정

## 1단계: Databricks 계정 생성

<Steps>
  <Step title="회원가입 페이지 접속">
    [databricks.com](https://databricks.com)에 접속한 뒤 **무료로 시작하기** 또는 **Try Databricks** 버튼을 클릭합니다.

    Community Edition을 사용하려면 [community.cloud.databricks.com](https://community.cloud.databricks.com)에 직접 접속해 **Sign Up**을 클릭합니다.
  </Step>
  <Step title="이메일 및 비밀번호 입력">
    이름, 이메일 주소, 비밀번호를 입력합니다. 기업 이메일을 사용하면 나중에 워크스페이스 관리와 SSO 설정이 더 편리합니다.
  </Step>
  <Step title="플랜 선택">
    - **Community Edition**: 무료, 단일 노드 클러스터, 학습용
    - **Premium Trial**: 14일 무료 체험, 전체 기능 사용 가능
    - **Enterprise**: 영업팀 문의 필요

    처음이라면 Community Edition 또는 Premium Trial을 선택합니다.
  </Step>
  <Step title="이메일 인증">
    가입 시 입력한 이메일 주소로 인증 메일이 발송됩니다. 메일을 열어 **이메일 인증** 링크를 클릭합니다.
  </Step>
</Steps>

## 2단계: 워크스페이스 설정

워크스페이스는 Databricks 작업의 기본 단위입니다. 클러스터, 노트북, 테이블, 워크플로우 등 모든 리소스가 워크스페이스 안에 존재합니다.

<Steps>
  <Step title="클라우드 및 리전 선택">
    유료 계정의 경우 클라우드(AWS, Azure, GCP)와 리전을 선택합니다. 데이터가 저장된 클라우드와 동일한 클라우드·리전을 선택하면 데이터 전송 비용을 줄일 수 있습니다.

    <Tip>
      한국 사용자라면 AWS `ap-northeast-2` (서울), Azure `koreacentral`, GCP `asia-northeast3` (서울)을 권장합니다.
    </Tip>
  </Step>
  <Step title="워크스페이스 이름 설정">
    워크스페이스 이름을 입력합니다. 이름은 나중에 변경할 수 없으므로 팀이나 프로젝트를 식별할 수 있는 명확한 이름을 사용하세요. 예: `my-team-prod`, `data-science-dev`.
  </Step>
  <Step title="워크스페이스 생성 완료 대기">
    워크스페이스 프로비저닝에는 약 5~10분이 소요됩니다. 완료되면 이메일 알림이 발송됩니다.
  </Step>
  <Step title="워크스페이스 접속">
    프로비저닝이 완료된 후 제공되는 URL(`https://<workspace-id>.azuredatabricks.net` 형식 등)로 접속합니다. 가입 시 사용한 이메일과 비밀번호, 또는 SSO로 로그인합니다.
  </Step>
</Steps>

## 3단계: 첫 번째 클러스터 생성

클러스터는 노트북 코드와 작업을 실행하는 컴퓨트 환경입니다. 학습 목적이라면 소규모 클러스터로 시작합니다.

<Steps>
  <Step title="컴퓨트 메뉴 접속">
    왼쪽 사이드바에서 **컴퓨트(Compute)** 아이콘을 클릭한 뒤 **클러스터 생성(Create cluster)**을 클릭합니다.
  </Step>
  <Step title="클러스터 기본 설정">
    아래 설정값을 입력합니다.

    | 항목 | 권장값 | 설명 |
    |---|---|---|
    | 클러스터 이름 | `my-first-cluster` | 구분하기 쉬운 이름 |
    | 클러스터 모드 | Single Node | 학습용 소규모 클러스터 |
    | Databricks Runtime | 최신 LTS 버전 (예: 15.4 LTS) | 안정성을 위해 LTS 선택 |
    | 노드 타입 | `m5.xlarge` (AWS) 또는 동급 | 4 vCPU, 16GB RAM |
    | 자동 종료 | 30분 | 비용 절감을 위해 설정 필수 |
  </Step>
  <Step title="클러스터 시작">
    **클러스터 생성(Create cluster)**을 클릭합니다. 클러스터 시작에는 약 3~5분이 소요됩니다. 상태가 **실행 중(Running)**으로 변경되면 사용 준비가 완료된 것입니다.

    <Warning>
      클러스터를 사용하지 않을 때는 반드시 **자동 종료**를 설정하거나 수동으로 종료하세요. 클러스터가 실행 중이면 비용이 계속 발생합니다.
    </Warning>
  </Step>
</Steps>

## 4단계: 첫 번째 노트북 실행

노트북은 코드, 텍스트, 시각화를 하나의 문서에서 작성하는 Databricks의 핵심 작업 도구입니다.

<Steps>
  <Step title="새 노트북 생성">
    왼쪽 사이드바에서 **새로 만들기(New)** → **노트북(Notebook)**을 클릭합니다. 노트북 이름을 `첫번째-노트북`으로 입력하고, 언어를 **Python**으로 선택한 뒤 앞서 생성한 클러스터를 연결합니다.
  </Step>
  <Step title="SparkSession 확인">
    노트북의 첫 번째 셀에 아래 코드를 입력하고 `Shift + Enter`로 실행합니다.

    ```python
    # SparkSession이 자동으로 생성되어 있는지 확인합니다
    print(spark)
    print(f"Spark 버전: {spark.version}")
    print(f"애플리케이션 이름: {spark.sparkContext.appName}")
    ```

    출력 결과에 SparkSession 정보가 나타나면 클러스터가 정상 연결된 것입니다.
  </Step>
  <Step title="샘플 데이터 로드">
    Databricks에는 학습용 샘플 데이터셋이 내장되어 있습니다. 다음 셀에 아래 코드를 입력하고 실행합니다.

    ```python
    # Databricks 내장 샘플 데이터 로드
    df = spark.read.format("csv") \
        .option("header", "true") \
        .option("inferSchema", "true") \
        .load("/databricks-datasets/samples/population-vs-price/data_geo.csv")

    # 데이터 형태 확인
    print(f"행 수: {df.count()}")
    print(f"컬럼: {df.columns}")
    df.printSchema()
    ```
  </Step>
  <Step title="기본 분석 실행">
    다음 셀에서 기본 데이터 분석을 수행합니다.

    ```python
    # 상위 5개 행 표시
    display(df.limit(5))
    ```

    ```python
    # 기초 통계 확인
    df.describe().show()
    ```

    ```python
    # SQL로도 동일한 작업 가능 — 임시 뷰 생성 후 쿼리
    df.createOrReplaceTempView("population_data")
    ```

    ```sql
    -- SQL 셀에서 실행 (셀 상단에 %sql 입력)
    %sql
    SELECT
      `2014 rank` AS rank,
      City,
      State,
      `2014 Population estimate` AS population
    FROM population_data
    ORDER BY population DESC
    LIMIT 10
    ```
  </Step>
  <Step title="Delta 테이블로 저장">
    분석한 데이터를 Delta 형식으로 저장합니다.

    ```python
    # Delta 테이블로 저장
    df.write.format("delta") \
        .mode("overwrite") \
        .saveAsTable("my_first_delta_table")

    print("Delta 테이블 저장 완료!")
    ```

    ```sql
    %sql
    -- 저장된 테이블 확인
    SELECT * FROM my_first_delta_table LIMIT 5;
    ```
  </Step>
</Steps>

## 기본 PySpark 패턴

아래는 Databricks에서 자주 사용하는 기본 PySpark 패턴입니다. 참고용으로 활용하세요.

<CodeGroup>

```python 데이터 읽기
# CSV 읽기
df_csv = spark.read.format("csv") \
    .option("header", "true") \
    .option("inferSchema", "true") \
    .load("dbfs:/path/to/file.csv")

# JSON 읽기
df_json = spark.read.format("json") \
    .load("dbfs:/path/to/file.json")

# Delta 테이블 읽기
df_delta = spark.read.format("delta") \
    .load("dbfs:/path/to/delta/table")

# 카탈로그 테이블 읽기 (Unity Catalog)
df_table = spark.table("my_catalog.my_schema.my_table")
```

```python 데이터 변환
from pyspark.sql import functions as F

# 컬럼 선택 및 필터
df_filtered = df.select("city", "population") \
    .filter(F.col("population") > 100000)

# 컬럼 추가
df_with_new_col = df.withColumn(
    "population_millions",
    F.col("population") / 1_000_000
)

# 그룹화 및 집계
df_agg = df.groupBy("state") \
    .agg(
        F.sum("population").alias("total_population"),
        F.count("*").alias("city_count")
    ) \
    .orderBy(F.desc("total_population"))
```

```python 데이터 저장
# Delta 형식으로 저장 (권장)
df.write.format("delta") \
    .mode("overwrite") \  # 또는 "append"
    .partitionBy("year", "month") \
    .saveAsTable("my_catalog.my_schema.my_table")

# 외부 경로에 저장
df.write.format("delta") \
    .mode("overwrite") \
    .save("abfss://container@storage.dfs.core.windows.net/path")
```

```sql SQL 쿼리
%sql
-- 테이블 생성
CREATE TABLE IF NOT EXISTS my_catalog.my_schema.sales (
  sale_id   BIGINT,
  product   STRING,
  amount    DOUBLE,
  sale_date DATE
)
USING DELTA;

-- 데이터 삽입
INSERT INTO my_catalog.my_schema.sales VALUES
  (1, 'Product A', 150.0, '2025-01-15'),
  (2, 'Product B', 230.5, '2025-01-16');

-- 집계 쿼리
SELECT
  product,
  SUM(amount) AS total_sales,
  COUNT(*) AS transaction_count
FROM my_catalog.my_schema.sales
GROUP BY product
ORDER BY total_sales DESC;
```

</CodeGroup>

## 자주 발생하는 문제

<AccordionGroup>
  <Accordion title="클러스터 연결이 되지 않아요">
    노트북 상단의 클러스터 드롭다운에서 생성한 클러스터를 선택했는지 확인합니다. 클러스터 상태가 **종료됨(Terminated)**이라면 클러스터 이름 옆의 **시작** 버튼을 클릭해 재시작합니다. 시작에는 약 3~5분이 소요됩니다.
  </Accordion>
  <Accordion title="셀 실행 시 'Table not found' 오류가 발생해요">
    Unity Catalog가 활성화된 워크스페이스에서는 테이블 이름에 `카탈로그.스키마.테이블` 형식을 사용해야 합니다. 또는 `USE CATALOG my_catalog; USE SCHEMA my_schema;` 명령으로 기본 카탈로그와 스키마를 설정하세요.
  </Accordion>
  <Accordion title="Community Edition에서 클러스터가 종료돼요">
    Community Edition에서는 2시간 비활성 후 클러스터가 자동 종료됩니다. 이는 Community Edition의 정책으로, 유료 플랜으로 업그레이드하면 자동 종료 시간을 조정할 수 있습니다.
  </Accordion>
  <Accordion title="DBFS 경로와 Unity Catalog 경로의 차이는 무엇인가요?">
    `dbfs:/` 경로는 워크스페이스 로컬 파일 시스템이고, Unity Catalog 테이블은 클라우드 오브젝트 스토리지에 저장됩니다. 신규 프로젝트는 Unity Catalog를 사용하는 것을 권장합니다. Unity Catalog는 중앙화된 거버넌스, 계보 추적, 세분화된 접근 제어를 제공합니다.
  </Accordion>
</AccordionGroup>
