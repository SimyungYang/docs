Databricks Workflows는 데이터 파이프라인, ML 학습, DLT 파이프라인을 하나의 Job으로 묶어 오케스트레이션하는 네이티브 서비스입니다. Apache Airflow나 외부 스케줄러 없이도 복잡한 DAG 형태의 태스크 의존성을 정의하고, 실패 시 자동 재시도와 Slack·이메일 알림을 구성할 수 있습니다.

## 핵심 개념

<CardGroup cols={3}>
  <Card title="Job" icon="briefcase">
    오케스트레이션의 최상위 단위입니다. 하나 이상의 태스크를 포함하며, 스케줄·트리거·권한·클러스터 설정을 관리합니다.
  </Card>
  <Card title="Task" icon="list-check">
    실제 작업을 수행하는 단위입니다. Notebook, Python 스크립트, JAR, SQL, DLT 파이프라인, dbt 프로젝트 등 다양한 유형을 지원합니다.
  </Card>
  <Card title="Task Dependency" icon="diagram-project">
    태스크 간 선후관계를 `depends_on`으로 정의합니다. 병렬 실행과 순차 실행을 자유롭게 조합하여 DAG를 구성합니다.
  </Card>
</CardGroup>

## Job 생성 방법

<Tabs>
  <Tab title="Databricks Asset Bundles (YAML)">
    코드로 Job을 정의하고 Git으로 버전 관리합니다. CI/CD 파이프라인과 자연스럽게 통합됩니다.

    ```yaml
    # databricks.yml
    bundle:
      name: sales-pipeline

    resources:
      jobs:
        daily_sales_etl:
          name: "Daily Sales ETL Pipeline"
          description: "매일 오전 6시 매출 데이터 처리"

          # 스케줄: 매일 UTC 21:00 (한국 기준 오전 6시)
          schedule:
            quartz_cron_expression: "0 0 21 * * ?"
            timezone_id: "UTC"
            pause_status: UNPAUSED

          # 실패 시 이메일 알림
          email_notifications:
            on_failure:
              - data-team@company.com
            on_success:
              - data-team@company.com

          # 재시도 정책
          max_concurrent_runs: 1

          tasks:
            # Task 1: Bronze 수집
            - task_key: ingest_raw_data
              description: "Auto Loader로 원시 데이터 수집"
              notebook_task:
                notebook_path: ./notebooks/01_bronze_ingest
                base_parameters:
                  date: "{{job.start_time.iso_date}}"
              new_cluster:
                spark_version: "15.4.x-scala2.12"
                node_type_id: "i3.xlarge"
                num_workers: 2

            # Task 2: Silver 정제 (Bronze 완료 후 실행)
            - task_key: transform_silver
              description: "데이터 정제 및 품질 검증"
              depends_on:
                - task_key: ingest_raw_data
              notebook_task:
                notebook_path: ./notebooks/02_silver_transform
              job_cluster_key: shared_cluster

            # Task 3: Gold 집계 (Silver 완료 후 실행)
            - task_key: aggregate_gold
              description: "BI용 집계 테이블 생성"
              depends_on:
                - task_key: transform_silver
              notebook_task:
                notebook_path: ./notebooks/03_gold_aggregate
              job_cluster_key: shared_cluster

            # Task 4, 5: 병렬 실행 (Gold 완료 후)
            - task_key: update_dashboard
              depends_on:
                - task_key: aggregate_gold
              notebook_task:
                notebook_path: ./notebooks/04_refresh_dashboard
              job_cluster_key: shared_cluster

            - task_key: send_report
              depends_on:
                - task_key: aggregate_gold
              notebook_task:
                notebook_path: ./notebooks/05_send_report
              job_cluster_key: shared_cluster

          job_clusters:
            - job_cluster_key: shared_cluster
              new_cluster:
                spark_version: "15.4.x-scala2.12"
                node_type_id: "i3.xlarge"
                num_workers: 4
                autoscale:
                  min_workers: 2
                  max_workers: 8
    ```
  </Tab>
  <Tab title="Databricks SDK (Python)">
    ```python
    from databricks.sdk import WorkspaceClient
    from databricks.sdk.service import jobs

    w = WorkspaceClient()

    created_job = w.jobs.create(
        name="Daily Sales ETL Pipeline",
        schedule=jobs.CronSchedule(
            quartz_cron_expression="0 0 21 * * ?",
            timezone_id="UTC",
            pause_status=jobs.PauseStatus.UNPAUSED,
        ),
        email_notifications=jobs.JobEmailNotifications(
            on_failure=["data-team@company.com"],
        ),
        tasks=[
            jobs.Task(
                task_key="ingest_raw_data",
                notebook_task=jobs.NotebookTask(
                    notebook_path="/Workspace/pipelines/01_bronze_ingest",
                    base_parameters={"date": "{{job.start_time.iso_date}}"},
                ),
                new_cluster=jobs.ClusterSpec(
                    spark_version="15.4.x-scala2.12",
                    node_type_id="i3.xlarge",
                    num_workers=2,
                ),
            ),
            jobs.Task(
                task_key="transform_silver",
                depends_on=[jobs.TaskDependency(task_key="ingest_raw_data")],
                notebook_task=jobs.NotebookTask(
                    notebook_path="/Workspace/pipelines/02_silver_transform",
                ),
                job_cluster_key="shared_cluster",
            ),
            jobs.Task(
                task_key="aggregate_gold",
                depends_on=[jobs.TaskDependency(task_key="transform_silver")],
                notebook_task=jobs.NotebookTask(
                    notebook_path="/Workspace/pipelines/03_gold_aggregate",
                ),
                job_cluster_key="shared_cluster",
            ),
        ],
        job_clusters=[
            jobs.JobCluster(
                job_cluster_key="shared_cluster",
                new_cluster=jobs.ClusterSpec(
                    spark_version="15.4.x-scala2.12",
                    node_type_id="i3.xlarge",
                    autoscale=jobs.AutoScale(min_workers=2, max_workers=8),
                ),
            )
        ],
    )

    print(f"Job 생성 완료 — Job ID: {created_job.job_id}")
    ```
  </Tab>
</Tabs>

## Task 유형별 사용 예시

<Accordion title="Notebook Task — 파라미터 전달">
  Job 파라미터를 노트북 위젯으로 전달합니다. 노트북에서 `dbutils.widgets.get()`으로 값을 수신합니다.

  ```python
  # 노트북 내부 코드
  import dbutils

  # Job에서 전달된 파라미터 수신
  date       = dbutils.widgets.get("date")
  env        = dbutils.widgets.get("env")       # dev / staging / prod
  source_path = dbutils.widgets.get("source_path")

  print(f"처리 날짜: {date}, 환경: {env}")

  # 실제 처리 로직
  df = spark.read.format("delta").load(f"{source_path}/{date}/")
  # ... 변환 로직 ...

  # 다음 태스크로 값 전달 (Task Values)
  dbutils.jobs.taskValues.set(key="record_count", value=df.count())
  ```
</Accordion>

<Accordion title="Python Script Task — 외부 패키지 활용">
  ```python
  # pipeline/silver_transform.py
  import sys
  from pyspark.sql import SparkSession
  from pyspark.sql.functions import col, current_timestamp

  def main():
      spark = SparkSession.builder.appName("SilverTransform").getOrCreate()

      # 이전 태스크의 결과값 읽기
      record_count = int(sys.argv[1]) if len(sys.argv) > 1 else 0
      print(f"Bronze에서 수신한 레코드 수: {record_count:,}")

      df = spark.table("bronze.raw_orders")
      silver_df = df.filter(col("amount") > 0).withColumn("updated_at", current_timestamp())
      silver_df.write.format("delta").mode("overwrite").saveAsTable("silver.orders")

  if __name__ == "__main__":
      main()
  ```
</Accordion>

## 재시도 정책과 알림 설정

```yaml
tasks:
  - task_key: ingest_raw_data
    notebook_task:
      notebook_path: ./notebooks/01_bronze_ingest

    # 재시도: 최대 3회, 5분 간격
    max_retries: 3
    min_retry_interval_millis: 300000  # 5분

    # 태스크 타임아웃 (초 단위)
    timeout_seconds: 3600  # 1시간

    # 태스크별 알림
    email_notifications:
      on_failure:
        - oncall@company.com
      on_start: []
      on_success: []
```

<Note>
  Job 수준의 `email_notifications`는 전체 Job의 성공/실패에 반응하고, 태스크 수준 설정은 개별 태스크에 반응합니다. 중요한 태스크는 태스크 수준 알림을 별도로 설정하는 것을 권장합니다.
</Note>

## 전체 파이프라인 구축 단계

<Steps>
  <Step title="태스크 분해와 의존성 정의">
    처리할 작업을 원자적 태스크로 분해합니다. 각 태스크가 단일 책임을 갖도록 설계하고, 태스크 간 데이터 흐름과 의존성을 그래프로 먼저 그려 보십시오. 병렬 실행 가능한 태스크를 식별하면 전체 실행 시간을 단축할 수 있습니다.
  </Step>
  <Step title="클러스터 전략 결정">
    모든 태스크가 공유 클러스터(Job Cluster)를 사용할지, 각자 전용 클러스터를 사용할지 결정합니다. 메모리 집약적 태스크와 CPU 집약적 태스크를 분리하면 비용 효율이 높아집니다.

    ```yaml
    job_clusters:
      - job_cluster_key: heavy_cluster
        new_cluster:
          node_type_id: "r6i.4xlarge"   # 메모리 최적화
          num_workers: 8
      - job_cluster_key: light_cluster
        new_cluster:
          node_type_id: "i3.xlarge"     # 범용
          num_workers: 2
    ```
  </Step>
  <Step title="파라미터와 Task Values 설계">
    Job 파라미터(`base_parameters`)로 날짜·환경 등 실행 맥락을 주입합니다. 태스크 간에 중간 결과를 전달할 때는 `dbutils.jobs.taskValues`를 사용합니다.

    ```python
    # 업스트림 태스크: 값 설정
    dbutils.jobs.taskValues.set(key="row_count", value=silver_df.count())

    # 다운스트림 태스크: 값 읽기
    row_count = dbutils.jobs.taskValues.get(
        taskKey="transform_silver",
        key="row_count",
        default=0,
    )
    print(f"수신된 레코드 수: {row_count:,}")
    ```
  </Step>
  <Step title="트리거 유형 선택">
    | 트리거 유형 | 사용 시나리오 |
    |------------|--------------|
    | Cron 스케줄 | 정해진 시간마다 실행 (일별 배치 등) |
    | 파일 도착 (File Arrival) | 특정 경로에 파일이 도착하면 자동 실행 |
    | Continuous | 완료 즉시 재시작 (Near Real-Time) |
    | 수동 / API | 온디맨드 또는 외부 시스템 호출 |
  </Step>
  <Step title="모니터링과 SLA 설정">
    Job 실행 시간이 SLA를 초과하면 알림을 보내도록 `health` 규칙을 설정합니다.

    ```yaml
    health:
      rules:
        - metric: RUN_DURATION_SECONDS
          op: GREATER_THAN
          value: 7200   # 2시간 초과 시 경고
    ```
  </Step>
</Steps>

## CLI로 Job 실행 및 모니터링

```bash
# Databricks CLI 설치
pip install databricks-cli

# Job 수동 실행
databricks jobs run-now --job-id 12345

# 파라미터와 함께 실행
databricks jobs run-now \
  --job-id 12345 \
  --notebook-params '{"date": "2024-03-15", "env": "prod"}'

# 실행 상태 확인
databricks runs get --run-id 98765

# 실행 취소
databricks runs cancel --run-id 98765

# 최근 실행 목록
databricks jobs list-runs --job-id 12345 --limit 10
```

## CI/CD 통합

```bash
# GitHub Actions 예시 — PR 머지 시 Job 자동 배포
# .github/workflows/deploy.yml

# Asset Bundle 유효성 검사
databricks bundle validate

# Staging 환경 배포
databricks bundle deploy --target staging

# Production 환경 배포
databricks bundle deploy --target production

# 배포된 Job 즉시 실행
databricks bundle run daily_sales_etl --target production
```

<Tip>
  Asset Bundles(`.databricks/bundle`)을 Git에 포함하면 Job 설정 변경도 PR로 리뷰할 수 있습니다. `databricks bundle validate`를 CI 파이프라인에 추가하면 YAML 오류를 배포 전에 잡을 수 있습니다.
</Tip>
