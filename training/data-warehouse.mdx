Databricks SQL Warehouse는 레이크하우스 아키텍처 위에서 엔터프라이즈급 SQL 워크로드를 실행하는 관리형 컴퓨팅 엔진입니다. 전용 SQL 엔진(Photon)이 내장되어 Delta 테이블에 대한 복잡한 분석 쿼리를 빠르게 처리하며, 여러 BI 도구와 JDBC/ODBC 표준 인터페이스로 연결됩니다.

## SQL Warehouse 유형 비교

## 데이터 웨어하우스 스키마 설계

### 스타 스키마 구성

```sql
-- 팩트 테이블: 주문 트랜잭션
CREATE TABLE IF NOT EXISTS gold.dw.fact_orders (
    order_key       BIGINT      GENERATED ALWAYS AS IDENTITY,
    order_id        STRING      NOT NULL,
    customer_key    BIGINT      NOT NULL,
    product_key     BIGINT      NOT NULL,
    date_key        INT         NOT NULL,   -- YYYYMMDD 형식
    region_key      INT         NOT NULL,
    quantity        INT,
    unit_price      DECIMAL(12, 2),
    discount_amount DECIMAL(12, 2),
    net_amount      DECIMAL(12, 2),
    order_ts        TIMESTAMP
)
USING DELTA
PARTITIONED BY (date_key)
TBLPROPERTIES (
    'delta.autoOptimize.optimizeWrite' = 'true',
    'delta.autoOptimize.autoCompact'   = 'true'
);

-- 차원 테이블: 고객
CREATE TABLE IF NOT EXISTS gold.dw.dim_customers (
    customer_key    BIGINT      GENERATED ALWAYS AS IDENTITY,
    customer_id     STRING      NOT NULL,
    customer_name   STRING,
    email           STRING,
    segment         STRING,    -- Enterprise / SMB / Individual
    country         STRING,
    city            STRING,
    scd_start_date  DATE,
    scd_end_date    DATE,      -- NULL이면 현재 유효한 레코드
    is_current      BOOLEAN    DEFAULT TRUE
)
USING DELTA;

-- 차원 테이블: 날짜
CREATE TABLE IF NOT EXISTS gold.dw.dim_date (
    date_key        INT         NOT NULL,
    full_date       DATE,
    year            INT,
    quarter         INT,
    month           INT,
    month_name      STRING,
    week_of_year    INT,
    day_of_week     INT,
    day_name        STRING,
    is_weekend      BOOLEAN,
    is_holiday      BOOLEAN,
    fiscal_year     INT,
    fiscal_quarter  INT
)
USING DELTA;
```

### 날짜 차원 테이블 자동 생성

```sql
-- 날짜 범위로 dim_date 채우기
INSERT INTO gold.dw.dim_date
SELECT
    CAST(DATE_FORMAT(date_val, 'yyyyMMdd') AS INT)    AS date_key,
    date_val                                           AS full_date,
    YEAR(date_val)                                     AS year,
    QUARTER(date_val)                                  AS quarter,
    MONTH(date_val)                                    AS month,
    DATE_FORMAT(date_val, 'MMMM')                     AS month_name,
    WEEKOFYEAR(date_val)                               AS week_of_year,
    DAYOFWEEK(date_val)                                AS day_of_week,
    DATE_FORMAT(date_val, 'EEEE')                     AS day_name,
    DAYOFWEEK(date_val) IN (1, 7)                     AS is_weekend,
    FALSE                                              AS is_holiday,
    CASE WHEN MONTH(date_val) >= 4
         THEN YEAR(date_val)
         ELSE YEAR(date_val) - 1
    END                                                AS fiscal_year,
    CASE
        WHEN MONTH(date_val) IN (4, 5, 6)   THEN 1
        WHEN MONTH(date_val) IN (7, 8, 9)   THEN 2
        WHEN MONTH(date_val) IN (10, 11, 12) THEN 3
        ELSE 4
    END                                                AS fiscal_quarter
FROM (
    SELECT EXPLODE(SEQUENCE(
        TO_DATE('2020-01-01'),
        TO_DATE('2030-12-31'),
        INTERVAL 1 DAY
    )) AS date_val
);
```

## 주요 분석 쿼리 패턴

### 매출 집계와 윈도우 함수

```sql
-- 분기별 매출 추이와 전분기 대비 성장률
WITH quarterly_sales AS (
    SELECT
        d.fiscal_year,
        d.fiscal_quarter,
        c.segment,
        SUM(f.net_amount)                   AS total_revenue,
        COUNT(DISTINCT f.order_id)          AS order_count,
        COUNT(DISTINCT f.customer_key)      AS unique_customers
    FROM gold.dw.fact_orders f
    JOIN gold.dw.dim_date     d ON f.date_key = d.date_key
    JOIN gold.dw.dim_customers c ON f.customer_key = c.customer_key
    WHERE d.fiscal_year >= 2023
    GROUP BY 1, 2, 3
),
growth AS (
    SELECT
        *,
        LAG(total_revenue) OVER (
            PARTITION BY segment
            ORDER BY fiscal_year, fiscal_quarter
        ) AS prev_quarter_revenue,
        ROUND(
            (total_revenue - LAG(total_revenue) OVER (
                PARTITION BY segment
                ORDER BY fiscal_year, fiscal_quarter
            )) / NULLIF(LAG(total_revenue) OVER (
                PARTITION BY segment
                ORDER BY fiscal_year, fiscal_quarter
            ), 0) * 100, 2
        ) AS qoq_growth_pct
    FROM quarterly_sales
)
SELECT
    fiscal_year,
    fiscal_quarter,
    segment,
    total_revenue,
    order_count,
    unique_customers,
    qoq_growth_pct
FROM growth
ORDER BY segment, fiscal_year, fiscal_quarter;
```

### 고객 세그먼트 분석 (RFM)

```sql
-- Recency-Frequency-Monetary 스코어 계산
WITH rfm_base AS (
    SELECT
        f.customer_key,
        c.customer_name,
        c.segment,
        MAX(d.full_date)                    AS last_order_date,
        COUNT(DISTINCT f.order_id)          AS frequency,
        SUM(f.net_amount)                   AS monetary
    FROM gold.dw.fact_orders f
    JOIN gold.dw.dim_customers c ON f.customer_key = c.customer_key AND c.is_current
    JOIN gold.dw.dim_date      d ON f.date_key = d.date_key
    WHERE d.full_date >= ADD_MONTHS(CURRENT_DATE(), -12)
    GROUP BY 1, 2, 3
),
rfm_scores AS (
    SELECT
        *,
        DATEDIFF(CURRENT_DATE(), last_order_date) AS recency_days,
        NTILE(5) OVER (ORDER BY DATEDIFF(CURRENT_DATE(), last_order_date) DESC) AS r_score,
        NTILE(5) OVER (ORDER BY frequency ASC)  AS f_score,
        NTILE(5) OVER (ORDER BY monetary ASC)   AS m_score
    FROM rfm_base
)
SELECT
    customer_key,
    customer_name,
    segment,
    recency_days,
    frequency,
    ROUND(monetary, 0) AS monetary,
    r_score, f_score, m_score,
    r_score + f_score + m_score AS rfm_total,
    CASE
        WHEN r_score >= 4 AND f_score >= 4 AND m_score >= 4 THEN '챔피언'
        WHEN r_score >= 3 AND f_score >= 3                  THEN '충성 고객'
        WHEN r_score >= 4                                    THEN '신규 고객'
        WHEN f_score >= 4 AND r_score <= 2                  THEN '위험 고객'
        ELSE '일반 고객'
    END AS customer_label
FROM rfm_scores
ORDER BY rfm_total DESC;
```

## SQL Warehouse 성능 최적화

<Steps>
  <Step title="Photon 엔진 활성화 확인">
    SQL Warehouse를 생성할 때 Photon이 기본 활성화됩니다. Photon은 Delta 테이블의 집계, 조인, 정렬 연산을 벡터화된 방식으로 처리하여 기존 Spark SQL 대비 최대 12배 빠른 성능을 제공합니다.
  </Step>
  <Step title="파티셔닝과 Z-Ordering 적용">
    쿼리에서 자주 사용되는 필터 컬럼을 기준으로 파티션을 구성하고, 고카디널리티 컬럼에 Z-Ordering을 적용합니다.

    ```sql
    -- 팩트 테이블 최적화
    OPTIMIZE gold.dw.fact_orders
    ZORDER BY (customer_key, product_key);
    ```
  </Step>
  <Step title="Materialized Views 활용">
    집계 계산이 복잡하거나 자주 조회되는 뷰를 Materialized View로 생성합니다. 소스 테이블이 업데이트될 때 자동으로 갱신됩니다.

    ```sql
    CREATE MATERIALIZED VIEW gold.dw.mv_monthly_revenue AS
    SELECT
        d.year,
        d.month,
        c.segment,
        SUM(f.net_amount)          AS total_revenue,
        COUNT(DISTINCT f.order_id) AS order_count
    FROM gold.dw.fact_orders f
    JOIN gold.dw.dim_date      d ON f.date_key = d.date_key
    JOIN gold.dw.dim_customers c ON f.customer_key = c.customer_key
    GROUP BY 1, 2, 3;
    ```
  </Step>
  <Step title="쿼리 캐시 설정">
    SQL Warehouse는 동일한 쿼리 결과를 자동으로 캐시합니다. 자주 사용하는 대시보드 쿼리는 스케줄 새로고침으로 캐시를 미리 준비해 두면 첫 로딩 속도가 개선됩니다.
  </Step>
  <Step title="Warehouse 크기 조정">
    쿼리 복잡도와 동시 사용자 수에 맞게 Warehouse 크기(X-Small ~ 4X-Large)를 조정합니다. Serverless Warehouse는 부하에 따라 자동으로 스케일 업/다운됩니다.

    | 크기 | 권장 동시 사용자 | 적합한 쿼리 유형 |
    |------|----------------|----------------|
    | X-Small | 1~5명 | 개발·테스트 |
    | Small | 5~10명 | 부서 분석 |
    | Medium | 10~30명 | 전사 대시보드 |
    | Large | 30~100명 | 대규모 BI |
  </Step>
</Steps>

## BI 도구 연동

<Tabs>
  <Tab title="Tableau">
    ```
    연결 설정:
    - 커넥터: Databricks
    - 서버 호스트: <workspace>.azuredatabricks.net
    - HTTP 경로: /sql/1.0/warehouses/<warehouse-id>
    - 인증: Personal Access Token (PAT)
    ```

    Tableau Desktop에서 **연결 → Databricks**를 선택하고 위 정보를 입력합니다. Unity Catalog가 활성화된 경우 카탈로그·스키마·테이블을 탐색하여 데이터 소스를 설정합니다.
  </Tab>
  <Tab title="Power BI">
    ```
    Power BI Desktop:
    1. 데이터 가져오기 → Azure Databricks 선택
    2. 서버 호스트: <workspace>.azuredatabricks.net
    3. HTTP 경로: /sql/1.0/warehouses/<warehouse-id>
    4. 가져오기 모드: DirectQuery (실시간 조회)
       또는 Import (데이터 복사)
    ```

    DirectQuery 모드를 사용하면 Delta 테이블의 최신 데이터를 항상 조회합니다. 대규모 테이블에는 집계 테이블이나 Materialized View를 먼저 구성하여 성능을 개선하십시오.
  </Tab>
  <Tab title="JDBC/ODBC">
    ```python
    # Python에서 JDBC로 연결 (예: 사내 애플리케이션)
    import jaydebeapi

    conn = jaydebeapi.connect(
        "com.databricks.client.jdbc.Driver",
        "jdbc:databricks://<workspace>.azuredatabricks.net:443"
        "/default;transportMode=http;ssl=1;"
        "httpPath=/sql/1.0/warehouses/<warehouse-id>;"
        "AuthMech=3;UID=token;PWD=<personal-access-token>",
    )

    cursor = conn.cursor()
    cursor.execute("SELECT * FROM gold.dw.mv_monthly_revenue LIMIT 100")
    rows = cursor.fetchall()
    ```
  </Tab>
</Tabs>

<Info>
  SQL Warehouse의 HTTP 경로와 호스트 주소는 Databricks 워크스페이스 → **SQL Warehouses** → 해당 Warehouse 선택 → **연결 세부 정보** 탭에서 확인할 수 있습니다.
