Lakebase는 Databricks 레이크하우스 플랫폼 위에서 OLTP(온라인 트랜잭션 처리) 워크로드를 처리하는 관리형 트랜잭션 데이터베이스 서비스입니다. PostgreSQL 호환 인터페이스를 제공하여 기존 애플리케이션을 코드 변경 없이 연결할 수 있으며, 데이터는 레이크하우스의 Delta 테이블로 저장되어 분석 파이프라인과 자연스럽게 통합됩니다.

## Lakebase가 해결하는 문제

전통적으로 운영 데이터베이스(OLTP)와 분석 플랫폼(OLAP)은 완전히 분리된 시스템으로 운영되었습니다. 데이터를 ETL로 옮기는 과정에서 지연이 발생하고, 두 시스템 간의 데이터 불일치가 잦았습니다. Lakebase는 레이크하우스 위에서 OLTP를 직접 실행하여 이 격차를 제거합니다.

<CardGroup cols={2}>
  <Card title="PostgreSQL 호환" icon="elephant">

  </Card>
  <Card title="레이크하우스 통합" icon="layer-group">
    트랜잭션 데이터가 Delta 테이블로 저장됩니다. ETL 없이 Databricks SQL, Spark, DLT 파이프라인에서 직접 트랜잭션 데이터를 분석할 수 있습니다.
  </Card>
  <Card title="Unity Catalog 거버넌스" icon="shield-check">

  <Card title="자동 확장" icon="up-right-and-down-left-from-center">

  </Card>
</CardGroup>

## 주요 사용 사례

<Tabs>
  <Tab title="AI 에이전트 메모리">
    LLM 에이전트의 대화 기록, 사용자 프로파일, 세션 상태를 Lakebase에 저장합니다. 에이전트가 종료되어도 컨텍스트가 보존되며, 분석 파이프라인이 에이전트 활동 데이터를 바로 집계할 수 있습니다.
  </Tab>
  <Tab title="애플리케이션 백엔드">
    웹/모바일 애플리케이션의 사용자 데이터, 주문, 재고를 Lakebase에 저장합니다. 운영 쿼리와 분석 쿼리가 동일한 데이터 위에서 실행되어 데이터 복제가 필요 없습니다.
  </Tab>
  <Tab title="실시간 피처 서빙">
    ML 모델 서빙에 필요한 최신 피처 값을 Lakebase에서 밀리초 수준 레이턴시로 조회합니다. Feature Store와 함께 사용하면 피처 계산 결과를 자동으로 동기화합니다.
  </Tab>
  <Tab title="이벤트 소싱">
    비즈니스 이벤트를 Lakebase에 append-only로 기록하고, DLT 파이프라인이 이벤트를 소비하여 집계 뷰를 실시간으로 업데이트합니다.
  </Tab>
</Tabs>

## 테이블 생성과 기본 DML

```sql
-- 세션 메모리 테이블 (AI 에이전트용)
CREATE TABLE IF NOT EXISTS agent_sessions (
    session_id      UUID        DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id         TEXT        NOT NULL,
    agent_type      TEXT        NOT NULL,
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    last_active_at  TIMESTAMPTZ DEFAULT NOW(),
    metadata        JSONB       DEFAULT '{}'
);

-- 대화 기록 테이블
CREATE TABLE IF NOT EXISTS conversation_messages (
    message_id   BIGSERIAL    PRIMARY KEY,
    session_id   UUID         NOT NULL REFERENCES agent_sessions(session_id) ON DELETE CASCADE,
    role         TEXT         NOT NULL CHECK (role IN ('user', 'assistant', 'system', 'tool')),
    content      TEXT         NOT NULL,
    tool_calls   JSONB,
    created_at   TIMESTAMPTZ  DEFAULT NOW()
);

-- 인덱스: 세션별 메시지 조회 성능
CREATE INDEX idx_messages_session
    ON conversation_messages (session_id, created_at DESC);

-- 피처 캐시 테이블 (ML 서빙용)
CREATE TABLE IF NOT EXISTS feature_cache (
    entity_id    TEXT        NOT NULL,
    feature_name TEXT        NOT NULL,
    feature_value DOUBLE PRECISION,
    computed_at  TIMESTAMPTZ NOT NULL,
    expires_at   TIMESTAMPTZ,
    PRIMARY KEY (entity_id, feature_name)
);

-- INSERT
INSERT INTO agent_sessions (user_id, agent_type, metadata)
VALUES ('user_001', 'sales-assistant', '{"language": "ko", "region": "KR"}')
RETURNING session_id, created_at;

-- UPDATE: 세션 마지막 활동 시간 갱신
UPDATE agent_sessions
SET    last_active_at = NOW()
WHERE  session_id = $1
RETURNING *;

-- UPSERT: 피처 값 갱신 (있으면 업데이트, 없으면 삽입)
INSERT INTO feature_cache (entity_id, feature_name, feature_value, computed_at, expires_at)
VALUES ('cust_001', 'churn_score', 0.73, NOW(), NOW() + INTERVAL '1 hour')
ON CONFLICT (entity_id, feature_name)
DO UPDATE SET
    feature_value = EXCLUDED.feature_value,
    computed_at   = EXCLUDED.computed_at,
    expires_at    = EXCLUDED.expires_at;
```

## Python 애플리케이션 연동

```python
import psycopg2
from psycopg2.extras import RealDictCursor, Json
import os
from contextlib import contextmanager
from datetime import datetime
from typing import Optional
import uuid

# 연결 설정
DB_CONFIG = {
    "host":     os.environ["LAKEBASE_HOST"],
    "port":     5432,
    "dbname":   os.environ["LAKEBASE_DATABASE"],
    "user":     "token",
    "password": os.environ["DATABRICKS_TOKEN"],
    "sslmode":  "require",
}

@contextmanager
def get_conn():
    conn = psycopg2.connect(**DB_CONFIG)
    try:
        yield conn
        conn.commit()
    except Exception:
        conn.rollback()
        raise
    finally:
        conn.close()


class AgentSessionManager:
    """AI 에이전트 세션 관리 클래스"""

    def create_session(self, user_id: str, agent_type: str, metadata: dict = None) -> str:
        with get_conn() as conn:
            with conn.cursor(cursor_factory=RealDictCursor) as cur:
                cur.execute(
                    """
                    INSERT INTO agent_sessions (user_id, agent_type, metadata)
                    VALUES (%s, %s, %s)
                    RETURNING session_id
                    """,
                    (user_id, agent_type, Json(metadata or {})),
                )
                row = cur.fetchone()
                return str(row["session_id"])

    def add_message(
        self,
        session_id: str,
        role: str,
        content: str,
        tool_calls: Optional[list] = None,
    ) -> int:
        with get_conn() as conn:
            with conn.cursor(cursor_factory=RealDictCursor) as cur:
                cur.execute(
                    """
                    INSERT INTO conversation_messages
                        (session_id, role, content, tool_calls)
                    VALUES (%s, %s, %s, %s)
                    RETURNING message_id
                    """,
                    (session_id, role, content, Json(tool_calls) if tool_calls else None),
                )
                # 세션 마지막 활동 시간 업데이트
                cur.execute(
                    "UPDATE agent_sessions SET last_active_at = NOW() WHERE session_id = %s",
                    (session_id,),
                )
                return cur.fetchone()["message_id"]

    def get_history(self, session_id: str, limit: int = 20) -> list[dict]:
        """최근 N개 메시지 조회"""
        with get_conn() as conn:
            with conn.cursor(cursor_factory=RealDictCursor) as cur:
                cur.execute(
                    """
                    SELECT role, content, tool_calls, created_at
                    FROM   conversation_messages
                    WHERE  session_id = %s
                    ORDER  BY created_at DESC
                    LIMIT  %s
                    """,
                    (session_id, limit),
                )
                rows = cur.fetchall()
                return list(reversed(rows))  # 오래된 순 정렬


# 사용 예시
mgr = AgentSessionManager()

session_id = mgr.create_session(
    user_id="user_001",
    agent_type="sales-assistant",
    metadata={"language": "ko"},
)
print(f"세션 생성: {session_id}")

mgr.add_message(session_id, "user",      "지난 분기 매출 현황을 알려주세요.")
mgr.add_message(session_id, "assistant", "지난 분기 총 매출은 12억 3,400만 원입니다.")

history = mgr.get_history(session_id)
for msg in history:
    print(f"[{msg['role']}] {msg['content']}")
```

## SQLAlchemy ORM 연동

```python
from sqlalchemy import create_engine, Column, String, Text, DateTime, func
from sqlalchemy.dialects.postgresql import UUID, JSONB
from sqlalchemy.orm import declarative_base, Session
import uuid

DATABASE_URL = (
    f"postgresql+psycopg2://token:{os.environ['DATABRICKS_TOKEN']}"
    f"@{os.environ['LAKEBASE_HOST']}:5432/{os.environ['LAKEBASE_DATABASE']}"
    f"?sslmode=require"
)

engine = create_engine(DATABASE_URL, pool_size=5, max_overflow=10)
Base   = declarative_base()


class AgentSession(Base):
    __tablename__ = "agent_sessions"

    session_id     = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    user_id        = Column(String,    nullable=False)
    agent_type     = Column(String,    nullable=False)
    created_at     = Column(DateTime(timezone=True), server_default=func.now())
    last_active_at = Column(DateTime(timezone=True), server_default=func.now())
    metadata_      = Column("metadata", JSONB, default={})


# 세션 생성
with Session(engine) as session:
    new_session = AgentSession(
        user_id="user_002",
        agent_type="data-analyst",
        metadata_={"timezone": "Asia/Seoul"},
    )
    session.add(new_session)
    session.commit()
    print(f"ORM 세션 생성: {new_session.session_id}")
```

## Databricks 분석 파이프라인과 통합

Lakebase의 데이터는 Databricks SQL에서 직접 쿼리할 수 있습니다. ETL 없이 운영 데이터를 실시간으로 분석합니다.

```python
# Databricks 노트북에서 Lakebase 직접 쿼리
lakebase_df = (
    spark.read
    .format("postgresql")
    .option("host",     "<instance-id>.lakebase.databricks.com")
    .option("port",     "5432")
    .option("database", "<database-name>")
    .option("dbtable",  "conversation_messages")
    .option("user",     "token")
    .option("password", dbutils.secrets.get("lakebase", "pat"))
    .load()
)

# 에이전트 활동 분석
from pyspark.sql.functions import col, count, date_trunc

daily_activity = (
    lakebase_df
    .withColumn("date", date_trunc("day", col("created_at")))
    .groupBy("date", "role")
    .agg(count("message_id").alias("message_count"))
    .orderBy("date", "role")
)

daily_activity.show()

# Delta 테이블로 저장하여 대시보드 연동
daily_activity.write.format("delta").mode("overwrite") \
    .saveAsTable("gold.agent_analytics.daily_activity")
```

<Info>
  Lakebase와 Databricks SQL Warehouse는 Unity Catalog를 공유합니다. 권한 설정도 동일한 GRANT 문으로 관리할 수 있어 거버넌스 정책을 일관되게 유지할 수 있습니다.
</Info>
