## 프로덕션 확장 시 고려할 사항

워크샵에서 구축한 프로토타입을 실제 서비스로 발전시키려면 다음 영역을 추가로 검토하세요.

### 성능 및 비용 최적화

```python
# 추천 결과 캐싱 예시 (Redis 연동)
import redis, json, hashlib

cache = redis.Redis(host="your-redis-host", port=6379, db=0)
CACHE_TTL = 3600  # 1시간

def get_recommendations_cached(user_id: str, top_n: int = 5) -> list:
    cache_key = hashlib.md5(f"{user_id}:{top_n}".encode()).hexdigest()
    cached = cache.get(cache_key)

    if cached:
        return json.loads(cached)

    recs = get_recommendations(user_id, top_n)
    cache.setex(cache_key, CACHE_TTL, json.dumps(recs, ensure_ascii=False))
    return recs
```

<Note>
  Model Serving 엔드포인트는 콜드 스타트(cold start) 지연이 발생할 수 있습니다. 트래픽이 예측 가능하다면 `scale_to_zero_enabled=False`로 설정하고 최소 1개 인스턴스를 유지하세요.
</Note>

### 모델 품질 모니터링

Databricks Lakehouse Monitoring을 사용해 추천 결과 품질을 지속적으로 추적합니다.

```python
from databricks.sdk import WorkspaceClient
from databricks.sdk.service.catalog import MonitorInferenceLog, MonitorInferenceLogProblemType

w = WorkspaceClient()

# 추론 로그 테이블 모니터링 설정
w.quality_monitors.create(
    table_name=f"{CATALOG}.{SCHEMA}.recommendation_logs",
    inference_log=MonitorInferenceLog(
        problem_type=MonitorInferenceLogProblemType.PROBLEM_TYPE_CLASSIFICATION,
        prediction_col="top_recommendation",
        timestamp_col="request_ts",
        model_id_col="model_version",
    ),
    output_schema_name=f"{CATALOG}.monitoring",
)
```

    ```python
    from databricks.sdk import WorkspaceClient

    w = WorkspaceClient()

    # 1. Model Serving 엔드포인트 삭제
    w.serving_endpoints.delete(name="smarttv-recommender-endpoint")

    # 2. 클러스터 종료 (삭제하지 않고 종료만)
    cluster_id = "<your-cluster-id>"
    w.clusters.delete(cluster_id=cluster_id)

    print("✓ 워크샵 리소스 정리 완료")
    ```

    <Warning>
      위 코드를 실행하면 엔드포인트와 클러스터가 삭제됩니다. 계속 실습하려면 실행하지 마세요.
