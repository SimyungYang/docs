워크샵을 원활하게 진행하려면 실습 시작 **전날까지** 이 페이지의 모든 설정을 완료하는 것을 권장합니다. 설정 도중 문제가 생기면 워크샵 Slack 채널(`#smarttv-workshop-help`)에 질문을 남겨 주세요.

## 사전 요구사항

<CardGroup cols={2}>
  <Card title="Databricks 계정" icon="database">
    Databricks Community Edition 또는 회사/행사 제공 Trial 워크스페이스 접근 권한이 필요합니다.
  </Card>
  <Card title="Python 3.10+" icon="python">
    로컬 머신에 Python 3.10 이상이 설치되어 있어야 합니다. 가상환경(venv 또는 conda) 사용을 권장합니다.
  </Card>
  <Card title="Git" icon="code-branch">
    워크샵 리포지터리를 클론하기 위해 Git이 필요합니다.
  </Card>
  <Card title="인터넷 연결" icon="wifi">
    Databricks Workspace 및 Model Serving 엔드포인트에 접근하기 위한 안정적인 인터넷 환경이 필요합니다.
  </Card>
</CardGroup>

<Warning>
  회사 네트워크를 사용하는 경우, 방화벽이 `*.azuredatabricks.net` 또는 `*.gcp.databricks.com` 도메인을 차단할 수 있습니다. IT 팀에 미리 확인하세요.
</Warning>

## 단계별 환경 설정

<Steps>
  <Step title="워크샵 리포지터리 클론">
    터미널을 열고 워크샵 GitHub 리포지터리를 클론합니다.

    ```bash
    git clone https://github.com/SimyungYang/genie-code-workshop-smarttv.git
    cd genie-code-workshop-smarttv
    ```

    클론이 완료되면 디렉터리 구조를 확인합니다.

    ```bash
    ls -la
    ```

    다음과 같은 구조가 보여야 합니다.

    ```
    genie-code-workshop-smarttv/
    ├── notebooks/          # Databricks 노트북 파일
    ├── src/                # Python 소스 코드
    ├── data/               # 샘플 데이터셋
    ├── genie-sdk/          # SmartTV Genie SDK 예제
    ├── requirements.txt
    └── README.md
    ```
  </Step>

  <Step title="Python 가상환경 생성 및 의존성 설치">
    프로젝트 루트에서 가상환경을 생성하고 필요한 패키지를 설치합니다.

    <Tabs>
      <Tab title="venv (표준)">
        ```bash
        python -m venv .venv
        source .venv/bin/activate      # macOS / Linux
        # .venv\Scripts\activate       # Windows PowerShell
        pip install --upgrade pip
        pip install -r requirements.txt
        ```
      </Tab>
      <Tab title="conda">
        ```bash
        conda create -n smarttv-workshop python=3.11 -y
        conda activate smarttv-workshop
        pip install -r requirements.txt
        ```
      </Tab>
    </Tabs>

    `requirements.txt`에 포함된 주요 패키지는 다음과 같습니다.

    ```text
    databricks-sdk>=0.20.0
    databricks-connect>=14.0
    mlflow>=2.12.0
    langchain>=0.2.0
    langchain-community>=0.2.0
    openai>=1.30.0
    pandas>=2.0.0
    pyarrow>=14.0.0
    requests>=2.31.0
    python-dotenv>=1.0.0
    ```
  </Step>

  <Step title="Databricks CLI 설치 및 인증 설정">
    Databricks CLI를 설치하고 워크스페이스에 인증합니다.

    ```bash
    pip install databricks-cli
    databricks configure --token
    ```

    프롬프트가 나타나면 다음 정보를 입력합니다.

    ```
    Databricks Host (should begin with https://): https://<your-workspace>.azuredatabricks.net
    Token: <your-personal-access-token>
    ```

    <Info>
      Personal Access Token은 Databricks Workspace → **User Settings** → **Developer** → **Access tokens** 에서 생성합니다. 토큰 만료일을 워크샵 날짜 이후로 설정하세요.
    </Info>

    인증이 제대로 됐는지 확인합니다.

    ```bash
    databricks clusters list
    ```

    클러스터 목록이 출력되면 인증 성공입니다.
  </Step>

  <Step title="환경 변수 파일 구성">
    프로젝트 루트에 `.env` 파일을 생성하고 필요한 변수를 설정합니다.

    ```bash
    cp .env.example .env
    ```

    `.env` 파일을 편집기로 열고 아래 값을 입력합니다.

    ```bash
    # Databricks 워크스페이스 설정
    DATABRICKS_HOST=https://<your-workspace>.azuredatabricks.net
    DATABRICKS_TOKEN=<your-personal-access-token>

    # 워크샵용 카탈로그 및 스키마
    CATALOG_NAME=smarttv_workshop
    SCHEMA_NAME=<your_name>_lab   # 예: gildong_lab

    # Model Serving 엔드포인트 (배포 후 입력)
    SERVING_ENDPOINT_NAME=smarttv-recommender-endpoint
    ```

    <Warning>
      `.env` 파일을 Git에 커밋하지 마세요. `.gitignore`에 이미 포함되어 있지만, 한 번 더 확인하세요.
    </Warning>
  </Step>

  <Step title="Databricks 워크스페이스에 노트북 가져오기">
    워크샵 노트북을 Databricks Workspace에 임포트합니다.

    ```bash
    databricks workspace import_dir ./notebooks /Users/<your-email>/smarttv-workshop
    ```

    또는 Databricks UI에서 직접 임포트할 수 있습니다.

    1. Databricks Workspace 왼쪽 사이드바에서 **Workspace** 를 클릭합니다.
    2. 원하는 폴더에서 **케밥 메뉴(⋮)** → **Import** 를 선택합니다.
    3. `notebooks/` 폴더의 `.ipynb` 파일을 드래그하거나 URL로 가져옵니다.
  </Step>

  <Step title="워크샵용 클러스터 생성">
    Databricks UI에서 워크샵 전용 클러스터를 생성합니다.

    1. 왼쪽 사이드바 **Compute** → **Create compute** 를 클릭합니다.
    2. 다음 설정을 적용합니다.

    | 항목 | 권장 값 |
    |------|---------|
    | 클러스터 이름 | `smarttv-workshop-<your-name>` |
    | Policy | **Unrestricted** |
    | Runtime | **14.3 LTS ML (Spark 3.5, Scala 2.12)** |
    | Worker type | `i3.xlarge` (AWS) 또는 `Standard_DS3_v2` (Azure) |
    | Min workers | 1 |
    | Max workers | 3 |
    | Autoscale | 활성화 |

    3. **Create compute** 를 클릭하고 클러스터 상태가 **Running** 이 될 때까지 기다립니다.

    <Note>
      ML Runtime을 선택해야 MLflow와 Feature Store 라이브러리가 미리 설치되어 있습니다. 일반 Runtime을 선택하면 일부 노트북이 동작하지 않을 수 있습니다.
    </Note>
  </Step>

  <Step title="설정 검증">
    모든 설정이 올바른지 확인하기 위해 검증 스크립트를 실행합니다.

    ```bash
    python src/check_setup.py
    ```

    정상적으로 설정됐을 때의 출력 예시입니다.

    ```
    ✓ Python 버전: 3.11.8
    ✓ Databricks SDK 연결 성공 (workspace: https://adb-xxxx.azuredatabricks.net)
    ✓ MLflow 연결 성공 (tracking URI: databricks)
    ✓ 카탈로그 접근 가능: smarttv_workshop
    ✓ 클러스터 상태 확인: smarttv-workshop-gildong (Running)
    ✓ 모든 사전 조건이 충족되었습니다. 워크샵을 시작할 준비가 되었습니다!
    ```

    <Warning>
      오류가 발생하면 해당 항목의 설정 단계로 돌아가 다시 확인하세요. `✗`로 표시된 항목의 오류 메시지를 Slack 채널에 공유하면 빠르게 도움을 받을 수 있습니다.
    </Warning>
  </Step>
</Steps>

## 자주 발생하는 문제

<AccordionGroup>
  <Accordion title="databricks clusters list 명령이 'Error: authentication error'를 반환합니다">
    토큰이 만료되었거나 호스트 URL이 잘못 입력됐을 가능성이 높습니다. `databricks configure --token` 을 다시 실행해 호스트와 토큰을 재입력하세요. 호스트 URL 끝에 슬래시(`/`)를 붙이지 마세요.
  </Accordion>
  <Accordion title="pip install 도중 'ERROR: Could not build wheels for pyarrow' 오류가 발생합니다">
    시스템에 C++ 빌드 도구가 없거나 Python 버전이 맞지 않을 때 발생합니다. `pip install --only-binary :all: pyarrow` 로 미리 빌드된 바이너리를 설치해 보세요. 해결되지 않으면 conda 환경을 대신 사용하세요.
  </Accordion>
  <Accordion title="클러스터 생성 후 상태가 'Error'로 표시됩니다">
    주로 워크스페이스 계정의 클러스터 생성 권한이 없을 때 발생합니다. 워크스페이스 관리자에게 `Cluster Create` 권한을 요청하거나, 강사가 미리 생성해둔 공유 클러스터를 사용하세요.
  </Accordion>
</AccordionGroup>

설정이 완료됐으면 [핸즈온 실습](/workshop/hands-on)으로 이동하세요.
